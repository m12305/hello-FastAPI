# 3.4 异步数据库操作

> FastAPI 是异步框架，但前三章我们一直在用同步的 SQLAlchemy——是时候让数据库层也用上 `async/await`，释放 FastAPI 的完整性能潜力。

---

## 1. 同步 vs 异步：为什么要关心？

### 同步模式下发生了什么？

```
请求 1 到达  ────────────────────────────────────────────────→ 响应 1
              │                    │
              ↓ db.execute()      ↑ 等数据库返回
              线程被阻塞（什么事都做不了，CPU 闲置）
              
请求 2 到达 ── 等待线程空闲 ─────────────────────────────────→ 响应 2
```

在同步模式下，当代码执行 `db.execute()` 去查询数据库时，**整个线程都被挂起**，等待数据库返回结果。这期间线程不能处理任何其他请求。

### 异步模式下发生了什么？

```
请求 1 到达  ──→ await db.execute() ── 挂起，释放线程 ──→ 结果回来 ──→ 响应 1

请求 2 到达  ──→ await db.execute() ── 挂起，释放线程 ──→ 响应 2
               （和请求 1 复用同一个线程！）
```

> **核心区别：** 同步是"打电话"（必须等对方说完才能做别的事），异步是"发微信"（发完去干别的，有消息了再回来看）。

---

## 2. 什么时候该用异步数据库？

| 场景 | 推荐 | 原因 |
|------|------|------|
| I/O 密集型 API（大量数据库查询） | ✅ 异步 | 最大化吞吐量 |
| CPU 密集型计算 | ❌ 同步足够 | 瓶颈在 CPU 不在 I/O |
| 简单的 CRUD | ⚠️ 看情况 | 请求量不大时同步足够 |
| SQLite（文件数据库） | ⚠️ 谨慎 | SQLite 异步支持有限 |
| MySQL / PostgreSQL | ✅ 异步 | 原生支持异步驱动 |

> **经验法则：** 如果你的 API 每秒请求数超过几十，或者数据库查询经常耗时超过 100ms——用异步。

## 4. 异步引擎与会话

对比同步版本的变化：

```python
"""database_async.py — 异步数据库基础设施"""
from sqlalchemy.ext.asyncio import (
    create_async_engine,      # ← sync: create_engine
    AsyncSession,             # ← sync: Session
    async_sessionmaker,       # ← sync: sessionmaker
    AsyncEngine,
)
from sqlalchemy.orm import DeclarativeBase

# ═════════ 异步数据库 URL ═════════
# SQLite（注意 driver 换成 aiosqlite）
DATABASE_URL = "sqlite+aiosqlite:///./app.db"

# MySQL（driver 换成 asyncmy）
# DATABASE_URL = "mysql+asyncmy://user:pass@localhost/dbname"

# ═════════ 异步引擎 ═════════
engine: AsyncEngine = create_async_engine(
    DATABASE_URL,
    echo=True,
    pool_size=20,          # 异步下连接池可以设大一些
    max_overflow=40,
    pool_pre_ping=True,    # 每次使用前检查连接是否存活
)

# ═════════ 异步 Session 工厂 ═════════
AsyncSessionLocal = async_sessionmaker(
    bind=engine,
    class_=AsyncSession,
    expire_on_commit=False,  # 异步下建议关闭
)


class Base(DeclarativeBase):
    pass


# ═════════ 异步依赖 ═════════
async def get_async_db():
    """异步数据库会话依赖——注意 async/await"""
    async with AsyncSessionLocal() as session:
        yield session  # yield 前打开，finally 后自动关闭
```

## 5. 异步 CRUD 操作

```python
"""crud_async.py — 异步 CRUD 操作"""
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy.orm import selectinload
from models import User, Post


# ════════════ Read ════════════
async def get_user(db: AsyncSession, user_id: int) -> User | None:
    """异步按 ID 查用户"""
    return await db.get(User, user_id)  # ← 加 await


async def get_users(db: AsyncSession, skip: int = 0, limit: int = 100) -> list[User]:
    """异步分页查用户"""
    stmt = select(User).offset(skip).limit(limit).order_by(User.id)
    result = await db.execute(stmt)          # ← await execute
    return list(result.scalars().all())


async def get_user_with_posts(db: AsyncSession, user_id: int) -> User | None:
    """异步 Eager Loading"""
    stmt = (
        select(User)
        .where(User.id == user_id)
        .options(selectinload(User.posts))
    )
    result = await db.execute(stmt)
    return result.scalars().first()


# ════════════ Create ════════════
async def create_user(db: AsyncSession, username: str, email: str) -> User:
    """异步创建用户"""
    user = User(username=username, email=email)
    db.add(user)
    await db.commit()          # ← await commit
    await db.refresh(user)     # ← await refresh
    return user


# ════════════ Update ════════════
async def update_user_email(db: AsyncSession, user_id: int, new_email: str) -> User:
    """异步更新用户邮箱"""
    user = await db.get(User, user_id)
    if user:
        user.email = new_email
        await db.commit()
        await db.refresh(user)
    return user


# ════════════ Delete ════════════
async def delete_user(db: AsyncSession, user_id: int) -> bool:
    """异步删除用户"""
    user = await db.get(User, user_id)
    if user:
        await db.delete(user)
        await db.commit()
        return True
    return False
```

## 6. 异步 FastAPI 端点

```python
"""main_async.py — 使用异步数据库的 FastAPI 应用"""
from fastapi import FastAPI, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from database_async import get_async_db, engine, Base
from schemas import UserCreate, UserResponse
import crud_async

app = FastAPI(title="Async Blog API")


# ═════════ 启动时建表 ═════════
@app.on_event("startup")
async def startup():
    """应用启动时异步创建表"""
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)


# ═════════ 路由 ═════════
@app.post("/users/", response_model=UserResponse)
async def create_user(
    user_in: UserCreate,
    db: AsyncSession = Depends(get_async_db),  # ← 注入异步 session
):
    """异步创建用户"""
    return await crud_async.create_user(db, user_in.username, user_in.email)


@app.get("/users/", response_model=list[UserResponse])
async def read_users(
    skip: int = 0,
    limit: int = 100,
    db: AsyncSession = Depends(get_async_db),
):
    """异步获取用户列表"""
    return await crud_async.get_users(db, skip=skip, limit=limit)


@app.get("/users/{user_id}", response_model=UserResponse)
async def read_user(
    user_id: int,
    db: AsyncSession = Depends(get_async_db),
):
    """异步获取单个用户"""
    user = await crud_async.get_user(db, user_id)
    if not user:
        raise HTTPException(status_code=404, detail="用户不存在")
    return user
```

## 7. 同步 vs 异步完整对比

| | 同步 | 异步 |
|------|------|------|
| **引擎** | `create_engine("sqlite:///...")` | `create_async_engine("sqlite+aiosqlite:///...")` |
| **Session** | `Session` / `sessionmaker` | `AsyncSession` / `async_sessionmaker` |
| **依赖** | `def get_db(): yield db` | `async def get_async_db(): async with ...: yield session` |
| **CRUD** | `session.get(User, 1)` | `await session.get(User, 1)` |
| **查询** | `session.scalars(stmt).all()` | `(await session.execute(stmt)).scalars().all()` |
| **提交** | `session.commit()` | `await session.commit()` |
| **端点** | `def endpoint(...):` | `async def endpoint(...):` |
| **数据库 URL** | `sqlite:///` / `mysql+pymysql://` | `sqlite+aiosqlite:///` / `mysql+asyncmy://` |

## 8. 连接池配置与优化

```python
engine = create_async_engine(
    DATABASE_URL,
    # ── 连接池 ──
    pool_size=20,            # 基础连接数
    max_overflow=40,         # 最大额外连接数
                            # 最大总连接 = pool_size + max_overflow = 60

    # ── 连接回收 ──
    pool_recycle=3600,       # 3600 秒后回收连接（避免数据库端断开）
    pool_pre_ping=True,      # 每次使用前 SELECT 1 测试连接是否存活

    # ── 超时 ──
    pool_timeout=30,         # 等待可用连接的最长时间（秒）

    # ── 性能 ──
    echo=False,              # 生产环境务必关闭
)
```

### MySQL 连接池调优公式

```
async_engine pool_size ≈ (CPU 核心数 * 2) + 有效磁盘数
max_overflow ≈ pool_size * 2

MySQL 端 max_connections 必须 ≥ 所有应用实例的总连接数
```

## 9. 混合使用同步和异步

现实中，你可能需要在异步应用中运行一些同步代码（比如旧模块、某些库）。

```python
from sqlalchemy.ext.asyncio import AsyncSession


async def get_user_stats(db: AsyncSession, user_id: int):
    """异步主流程中运行同步查询"""
    # 方式 1：run_sync —— 在异步 Session 中执行同步代码块
    sync_result = await db.run_sync(
        lambda session: session.execute(
            "SELECT COUNT(*) FROM posts WHERE user_id = :uid",
            {"uid": user_id}
        ).scalar()
    )
    return {"post_count": sync_result}
```

## 10. 性能对比：眼见为实

```python
"""benchmark_async.py — 异步 vs 同步性能对比"""
import time
import asyncio
import httpx


async def benchmark():
    """用 50 个并发请求对比两种模式"""
    async with httpx.AsyncClient(base_url="http://127.0.0.1:8000") as client:
        tasks = [client.get("/users/") for _ in range(50)]
        start = time.perf_counter()
        results = await asyncio.gather(*tasks)
        elapsed = time.perf_counter() - start

        success = sum(1 for r in results if r.status_code == 200)
        print(f"50 并发请求: {success}/50 成功, 耗时 {elapsed:.2f}s")
        print(f"平均 QPS: {50 / elapsed:.0f}")


asyncio.run(benchmark())
# 预期：异步版 QPS 大约是同步版的 2-5 倍
```
