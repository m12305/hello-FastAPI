# 3.1 SQLAlchemy 基础

> 从"用字典假装数据库"到真正的持久化存储——掌握 SQLAlchemy 2.0 的核心用法，为 FastAPI 集成打下坚实基础。

---

## 1. 为什么需要 ORM？

在前面两个阶段，我们一直用 Python 的 `dict` 或 `list` 来"模拟"数据库：

```python
# 阶段 1-2 的"数据库"——进程重启就没了
fake_db = {
    1: {"name": "张三", "email": "zhang@example.com"},
    2: {"name": "李四", "email": "li@example.com"},
}
```

**这种方式的问题：**

| 问题 | 表现 |
|------|------|
| **数据不持久** | 服务重启，所有数据丢失 |
| **无并发安全** | 多个请求同时写，数据会错乱 |
| **查询能力弱** | 无法做复杂筛选、排序、联表 |
| **无类型约束** | 字典可以随便塞，没有任何校验 |
| **无迁移机制** | 表结构变了，旧数据没法自动迁移 |

**ORM（Object-Relational Mapping）** 解决了这些问题——它让你用 Python 类来操作数据库，而不用直接写 SQL 字符串。

```
你写的代码                         ORM 做的事                         数据库
─────────────────────────────────────────────────────────────────────────────
user.name = "王五"    →    UPDATE users SET name='王五'   →   MySQL
db.add(user)                         WHERE id = 3;               SQLite
db.commit()                                                        PostgreSQL
```

> SQLAlchemy 是 Python 生态中最强大、最成熟的 ORM 框架。FastAPI 官方文档推荐使用它。

## 3. SQLAlchemy 核心架构

SQLAlchemy 有两个层面的 API：

```
┌─────────────────────────────────────────┐
│               ORM 层                     │
│  session.query(User).filter(...)        │  ← 你主要用这层
│  session.execute(select(User))          │
├─────────────────────────────────────────┤
│               Core 层                    │
│  engine.execute("SELECT * FROM users")  │  ← 底层 SQL 抽象
│  Table, MetaData, select()              │
├─────────────────────────────────────────┤
│               DBAPI                      │
│  sqlite3 / pymysql / asyncmy / ...       │  ← 真正和数据库通信
└─────────────────────────────────────────┘
```

> 本阶段聚焦 **ORM 层**，采用 SQLAlchemy 2.0 风格（`select()` + `session.execute()`），而非旧的 `session.query()` 风格。

---

## 4. 引擎（Engine）——连接数据库的入口

```python
from sqlalchemy import create_engine

# SQLite 引擎（文件存在则连接，不存在则创建）
engine = create_engine("sqlite:///./test.db", echo=True)
#                           │              │
#                           │              └── echo=True: 在控制台打印所有 SQL
#                           └── 使用 SQLite，db 文件在当目录

# 连接内存数据库（速度快，适合测试）
# engine = create_engine("sqlite:///:memory:")

# 连接 MySQL（后续章节）
# engine = create_engine("mysql+pymysql://user:pass@localhost/dbname")
```

| 参数 | 说明 | 常用值 |
|------|------|--------|
| `echo` | 打印 SQL 语句 | `True`（调试）/ `False`（生产） |
| `pool_size` | 连接池大小 | `5`（默认） |
| `max_overflow` | 超出 pool_size 的最大连接数 | `10`（默认） |

---

## 5. 会话（Session）——操作数据的单位

```python
from sqlalchemy.orm import Session, sessionmaker

# Session 工厂
SessionLocal = sessionmaker(bind=engine, autoflush=False, autocommit=False)

# 每次请求创建独立的 session
session = SessionLocal()

# ... 数据库操作 ...

session.close()  # 用完必须关（后面用 yield 依赖自动管理）
```

**Session 的生命周期（重要！）：**

```
请求到达 → 创建 Session → 操作数据库 → 提交/回滚 → 关闭 Session → 请求结束
```

> `autoflush=False`：手动控制何时将内存中的变更同步到数据库。`autocommit=False`：手动控制事务提交。在 Web 应用中，这两个都设为 `False` 是最安全的做法。

## 6. 定义模型——用 Python 类描述数据库表

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
from sqlalchemy import String, Integer, DateTime, func


# 第一步：定义基类
class Base(DeclarativeBase):
    """所有 ORM 模型的基类"""
    pass


# 第二步：定义模型
class User(Base):
    __tablename__ = "users"  # 表名

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    email: Mapped[str] = mapped_column(String(100), unique=True, nullable=False)
    age: Mapped[int | None] = mapped_column(default=None)
    is_active: Mapped[bool] = mapped_column(default=True)
    created_at: Mapped[datetime] = mapped_column(DateTime, server_default=func.now())

    def __repr__(self):
        """友好的打印输出"""
        return f"<User(id={self.id}, username='{self.username}')>"
```

### 核心概念对比

| 概念 | Python | 数据库 |
|------|--------|--------|
| **基类** `Base` | 所有模型的父类 | 对应整个数据库 Schema |
| **模型类** `User` | Python 类 | 对应一张表 `users` |
| **属性** `id`, `username` | 类属性（带类型注解） | 对应表中的列 |
| **实例** `user = User(...)` | 类的一个实例 | 对应表中的一行数据 |

### `mapped_column()` 常用参数

```python
# 完整参数示例
mapped_column(
    String(100),           # 列类型 + 长度
    primary_key=True,      # 主键
    unique=True,           # 唯一约束
    nullable=False,        # 不允许为 NULL
    default="default_val", # Python 层默认值
    server_default="now()",# 数据库层默认值（SQL 表达式）
    index=True,            # 创建索引
    autoincrement=True,    # 自增
)
```

## 7. 创建表——`Base.metadata.create_all()`

```python
# 根据所有继承 Base 的模型，在数据库中创建对应的表
Base.metadata.create_all(bind=engine)
```

> `create_all()` 只会创建**不存在的**表。已有表不会被修改或删除。这是开发阶段的方案——生产环境要用 Alembic（第 3.3 章）。

## 8. CRUD 操作——增删改查

### Create（增）

```python
# 方式 1：创建实例 → add → commit
user = User(username="张三", email="zhang@example.com", age=25)
session.add(user)
session.commit()   # 提交事务，数据真正写入数据库
session.refresh(user)  # 刷新实例，获取数据库生成的 id 和默认值
print(user.id)     # 现在有 id 了

# 方式 2：批量添加
users = [
    User(username="李四", email="li@example.com", age=30),
    User(username="王五", email="wang@example.com", age=28),
]
session.add_all(users)
session.commit()
```

### Read（查）

```python
from sqlalchemy import select

# 查单个——按主键
user = session.get(User, 1)  # 高效，直接查主键
print(user)  # <User(id=1, username='张三')>

# 查单个——按条件
stmt = select(User).where(User.username == "张三")
user = session.scalars(stmt).first()  # .first() 返回第一个或 None

# 查多个——全部
stmt = select(User).where(User.is_active == True)
users = session.scalars(stmt).all()  # .all() 返回 list

# 查多个——分页
stmt = select(User).offset(0).limit(10)
users = session.scalars(stmt).all()

# 常用筛选方法
stmt = select(User).where(
    User.age >= 18,
    User.is_active == True,
).order_by(User.created_at.desc())
```

> SQLAlchemy 2.0 推荐用 `select()` + `session.scalars()`，而不是旧的 `session.query(User)`。

### Update（改）

```python
# 方式 1：查出实例 → 修改属性 → commit
user = session.get(User, 1)
user.age = 26
user.email = "new_email@example.com"
session.commit()
session.refresh(user)

# 方式 2：批量更新（不先查出）
from sqlalchemy import update

stmt = update(User).where(User.is_active == False).values(is_active=True)
session.execute(stmt)
session.commit()
```

### Delete（删）

```python
# 方式 1：查出实例 → delete → commit
user = session.get(User, 3)
session.delete(user)
session.commit()

# 方式 2：批量删除
from sqlalchemy import delete

stmt = delete(User).where(User.is_active == False)
session.execute(stmt)
session.commit()
```

## 9. 事务管理

```python
# 正常提交
session = SessionLocal()
try:
    user = User(username="test", email="test@test.com")
    session.add(user)
    session.commit()  # 提交
except Exception as e:
    session.rollback()  # 出错回滚
    raise e
finally:
    session.close()  # 无论如何都要关
```

> 在 FastAPI 中，事务管理被依赖注入的 `yield` 模式自动化（第 3.2 章）。

## 10. 关系（Relationship）——表与表的关联

```python
from sqlalchemy import ForeignKey
from sqlalchemy.orm import relationship


class Post(Base):
    __tablename__ = "posts"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    title: Mapped[str] = mapped_column(String(200))
    content: Mapped[str] = mapped_column(String(5000))
    user_id: Mapped[int] = mapped_column(ForeignKey("users.id"))

    # 反向关系：通过 post.author 可以直接拿到 User 对象
    author: Mapped["User"] = relationship(back_populates="posts")


# 在 User 类中补充反向关系
User.posts: Mapped[list["Post"]] = relationship(back_populates="author")
```

使用：

```python
# 正向：从 Post 找到它的作者
post = session.get(Post, 1)
print(post.author.username)  # 自动查 User 表

# 反向：从 User 找到他所有的文章
user = session.get(User, 1)
for post in user.posts:
    print(post.title)
```

## 11. 完整演示脚本

```python
"""3.1-sqlalchemy-demo.py — SQLAlchemy 2.0 基础完整演示"""
from sqlalchemy import create_engine, String, Integer, DateTime, ForeignKey, select, func
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, Session, sessionmaker, relationship

# ═══════════════════════════════════════════════════════════
# 1. 引擎 & 会话
# ═══════════════════════════════════════════════════════════
engine = create_engine("sqlite:///./demo.db", echo=True)
SessionLocal = sessionmaker(bind=engine, autoflush=False, autocommit=False)

# ═══════════════════════════════════════════════════════════
# 2. 基类 & 模型
# ═══════════════════════════════════════════════════════════
class Base(DeclarativeBase):
    pass


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    email: Mapped[str] = mapped_column(String(100), unique=True, nullable=False)
    age: Mapped[int | None] = mapped_column(default=None)
    is_active: Mapped[bool] = mapped_column(default=True)
    created_at: Mapped[DateTime] = mapped_column(DateTime, server_default=func.now())

    posts: Mapped[list["Post"]] = relationship(back_populates="author")

    def __repr__(self):
        return f"<User(id={self.id}, username='{self.username}')>"


class Post(Base):
    __tablename__ = "posts"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    title: Mapped[str] = mapped_column(String(200))
    content: Mapped[str] = mapped_column(String(5000))
    user_id: Mapped[int] = mapped_column(ForeignKey("users.id"))

    author: Mapped["User"] = relationship(back_populates="posts")

    def __repr__(self):
        return f"<Post(id={self.id}, title='{self.title}')>"


# ═══════════════════════════════════════════════════════════
# 3. 建表
# ═══════════════════════════════════════════════════════════
Base.metadata.create_all(bind=engine)

# ═══════════════════════════════════════════════════════════
# 4. CRUD 演示
# ═══════════════════════════════════════════════════════════
def demo():
    session = SessionLocal()

    try:
        # ── Create ──
        user = User(username="张三", email="zhang@example.com", age=25)
        session.add(user)
        session.commit()
        session.refresh(user)
        print(f"✅ 创建用户: {user}")

        # ── Read ──
        stmt = select(User).where(User.username == "张三")
        found = session.scalars(stmt).first()
        print(f"✅ 查询用户: {found}")

        # ── Update ──
        found.age = 26
        session.commit()
        print(f"✅ 更新年龄: {found.age}")

        # ── Delete ──
        session.delete(found)
        session.commit()
        print(f"✅ 删除用户: {found}")

    except Exception as e:
        session.rollback()
        print(f"❌ 出错: {e}")
    finally:
        session.close()


if __name__ == "__main__":
    demo()
```

运行：

```bash
python 3.1-sqlalchemy-demo.py
```

## 12. SQLAlchemy 1.4 vs 2.0 写法对比

| 操作 | 1.4 旧风格（`session.query()`） | 2.0 新风格（`select()` + `session.execute()`） |
|------|----------------------------|------------------------------------------|
| 查询 | `session.query(User).filter(...)` | `session.scalars(select(User).where(...))` |
| 模型定义 | `declarative_base()` + `Column()` | `DeclarativeBase` + `Mapped` + `mapped_column()` |
| 类型提示 | 无 | 完整 `Mapped[type]` 支持 |

> **本系列全部采用 2.0 新风格**。如果遇到旧项目使用 `query()` 风格，知道它是旧写法即可。
