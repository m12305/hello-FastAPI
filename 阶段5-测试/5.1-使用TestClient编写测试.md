# 5.1 使用 TestClient 编写测试

> 不靠手动 curl，不靠 Postman 逐一点击——用 FastAPI 内置的 TestClient 为每个接口编写自动化测试，让测试成为你重构时的安全网。

---

## 1. 为什么写测试？

很多人觉得"我手动测过了，没问题"，直到：

```
某天你改了一个看似无关的函数 → 部署上线 → 半夜报警 → 用户数据丢失
                                    ↑
                        原来那个"无关"的函数是核心逻辑的基石
```

| 没有测试 | 有测试 |
|---------|--------|
| 改代码后手动点一遍 → 容易漏 | `pytest` 一键跑完 → 30 秒全量验证 |
| 半夜出问题不知道 | CI 自动跑，不通过不合并 |
| 不敢重构 | 改完立刻知道有没有破坏 |
| 新人接手看不懂逻辑 | 测试本身就是最好的文档 |

> 💡 测试不是额外的工作，测试是你的"安全网"和"活文档"。

---

## 2. TestClient 快速上手

FastAPI 继承自 Starlette，内置了基于 `httpx` 的 `TestClient`。**用它发请求就像真的在调 API，但它不会占用端口、不需要启动服务器。**

### 2.1 安装依赖

```bash
pip install httpx pytest
```

### 2.2 第一个测试

假设你有一个最简单的 FastAPI 应用：

```python
# main.py
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def root():
    return {"message": "Hello FastAPI"}

@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}
```

对应的测试文件：

```python
# test_main.py
from fastapi.testclient import TestClient
from main import app

client = TestClient(app)  # ← 创建测试客户端

def test_root():
    """测试根路径"""
    response = client.get("/")
    assert response.status_code == 200
    assert response.json() == {"message": "Hello FastAPI"}

def test_read_item():
    """测试路径参数"""
    response = client.get("/items/42")
    assert response.status_code == 200
    assert response.json() == {"item_id": 42}

def test_read_item_invalid():
    """测试非法参数——FastAPI 应返回 422"""
    response = client.get("/items/abc")  # item_id 要求 int
    assert response.status_code == 422
```

运行：

```bash
# 在 test_main.py 所在目录执行
pytest test_main.py -v

# 输出示例：
# test_main.py::test_root              PASSED
# test_main.py::test_read_item         PASSED
# test_main.py::test_read_item_invalid PASSED
```

### 2.3 TestClient 的核心方法

TestClient 的使用方式和 `httpx` 几乎一致——每个 HTTP 方法都有对应的函数：

```python
from fastapi.testclient import TestClient
from main import app

client = TestClient(app)

# ── GET ──
response = client.get("/users?page=1&size=10")       # 查询参数写 URL 上
response = client.get("/users", params={"page": 1})   # 或通过 params=

# ── POST（JSON 请求体）──
response = client.post(
    "/users",
    json={"username": "张三", "email": "zhang@example.com", "password": "12345678"}
)

# ── PUT / PATCH ──
response = client.put("/users/1", json={"username": "李四"})
response = client.patch("/users/1", json={"email": "lisi@example.com"})

# ── DELETE ──
response = client.delete("/users/1")

# ── 请求头 ──
response = client.get(
    "/me",
    headers={"Authorization": "Bearer eyJhbGciOi..."}
)

# ── 表单提交（OAuth2 密码模式）──
response = client.post(
    "/login",
    data={"username": "admin", "password": "admin123"}  # ← 用 data= 不是 json=
)

# ── 文件上传 ──
response = client.post(
    "/upload",
    files={"file": ("avatar.png", open("avatar.png", "rb"), "image/png")}
)
```

## 3. Response 对象：怎么断言？

```python
response = client.get("/users/1")

# ── 状态码 ──
response.status_code         # 200
response.is_success           # True（2xx）
response.is_client_error      # False（4xx）
response.is_server_error      # False（5xx）

# ── JSON 响应 ──
data = response.json()        # 将响应体解析为 dict
assert data["username"] == "张三"

# ── 响应头 ──
response.headers["content-type"]   # "application/json"

# ── 响应文本 ──
response.text                # 原始响应字符串

# ── 响应体（bytes）──
response.content             # b'...'
```

### 常用断言模式

```python
def test_create_user():
    response = client.post("/users", json={
        "username": "张三",
        "email": "zhang@example.com",
        "password": "12345678"
    })

    # 1. 检查状态码
    assert response.status_code == 201  # 或 200

    # 2. 检查返回数据包含期望字段
    data = response.json()
    assert data["username"] == "张三"
    assert data["email"] == "zhang@example.com"
    assert "id" in data          # 应有自动生成的 ID
    assert "password" not in data  # 绝对不能返回密码！

    # 3. 检查字段类型（可选但更严谨）
    assert isinstance(data["id"], int)

def test_duplicate_username():
    """重复用户名应返回 4xx"""
    # 先注册一个
    client.post("/users", json={
        "username": "张三", "email": "a@b.com", "password": "12345678"
    })
    # 再用相同用户名注册
    response = client.post("/users", json={
        "username": "张三", "email": "c@d.com", "password": "87654321"
    })
    assert response.status_code in (400, 409)  # 冲突错误
```

## 4. 测试数据库：隔离是关键

**绝对不要用生产数据库跑测试！** 测试必须能在任何环境复现，每次运行的结果一致。

### 4.1 SQLite 内存数据库

```python
# database.py  —— 测试友好版
import os
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, DeclarativeBase

TESTING = os.getenv("TESTING", "").lower() == "true"

# 测试环境用 SQLite 内存库，开发/生产用 MySQL
if TESTING:
    DATABASE_URL = "sqlite:///test.db"     # 文件库，方便调试
    # 或用纯内存库：DATABASE_URL = "sqlite:///:memory:"
else:
    DATABASE_URL = "mysql+pymysql://root:password@localhost:3306/mydb"

engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False} if "sqlite" in DATABASE_URL else {},
    echo=False,
)
TestingSessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

class Base(DeclarativeBase):
    pass

def get_db():
    db = TestingSessionLocal()
    try:
        yield db
    finally:
        db.close()
```

### 4.2 使用 pytest fixture 管理测试数据

```python
# conftest.py  —— 放在测试目录下，自动被 pytest 发现
import pytest
from fastapi.testclient import TestClient
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

from database import Base, get_db
from main import app

# ── 1. 创建测试引擎 ──
@pytest.fixture(scope="function")
def db():
    """每个测试函数使用独立的数据库"""
    # SQLite 内存库（":memory:" 会在连接关闭时清空）
    engine = create_engine(
        "sqlite:///:memory:",
        connect_args={"check_same_thread": False},
    )
    Base.metadata.create_all(bind=engine)  # 建表

    TestingSession = sessionmaker(bind=engine)
    session = TestingSession()
    try:
        yield session
    finally:
        session.close()
        Base.metadata.drop_all(bind=engine)  # 清理：删表

# ── 2. 创建 TestClient（注入测试数据库）──
@pytest.fixture(scope="function")
def client(db):
    """返回一个注入了测试数据库的 TestClient"""
    def override_get_db():
        try:
            yield db
        finally:
            pass

    app.dependency_overrides[get_db] = override_get_db
    with TestClient(app) as c:
        yield c
    app.dependency_overrides.clear()  # 清理覆盖
```

### 4.3 编写数据库相关测试

```python
# test_users.py
def test_create_user(client):
    """创建用户 → 验证返回数据"""
    response = client.post("/users", json={
        "username": "zhangsan",
        "email": "zhang@example.com",
        "password": "secret123"
    })
    assert response.status_code == 200
    data = response.json()
    assert data["username"] == "zhangsan"
    assert "password" not in data

def test_list_users(client):
    """查询用户列表"""
    # Arrange: 先创建两个用户
    client.post("/users", json={"username": "u1", "email": "u1@x.com", "password": "pwd"})
    client.post("/users", json={"username": "u2", "email": "u2@x.com", "password": "pwd"})

    # Act
    response = client.get("/users")

    # Assert
    assert response.status_code == 200
    data = response.json()
    assert len(data) >= 2

def test_get_user_not_found(client):
    """查询不存在的用户 → 404"""
    response = client.get("/users/99999")
    assert response.status_code == 404

def test_update_user(client):
    """更新用户信息"""
    # 先创建
    r = client.post("/users", json={"username": "old", "email": "old@x.com", "password": "pwd"})
    user_id = r.json()["id"]

    # 再更新
    response = client.patch(f"/users/{user_id}", json={"username": "new_name"})
    assert response.status_code == 200
    assert response.json()["username"] == "new_name"

def test_delete_user(client):
    """删除用户 → 删除后查不到"""
    r = client.post("/users", json={"username": "tmp", "email": "tmp@x.com", "password": "pwd"})
    user_id = r.json()["id"]

    response = client.delete(f"/users/{user_id}")
    assert response.status_code == 200

    # 确认已删除
    response = client.get(f"/users/{user_id}")
    assert response.status_code == 404
```

## 5. 测试文件组织

```
项目根目录/
├── app/
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── routers/
│       ├── users.py
│       └── posts.py
├── tests/                      # ← 测试目录，与 app/ 平级
│   ├── conftest.py             #   共享 fixture
│   ├── test_users.py           #   用户相关测试
│   ├── test_posts.py           #   文章相关测试
│   └── test_auth.py            #   认证相关测试
└── requirements.txt
```

运行：

```bash
# 运行全部测试
pytest tests/ -v

# 只运行某个文件
pytest tests/test_users.py -v

# 只运行名称包含 "create" 的测试
pytest tests/ -v -k "create"

# 显示打印输出
pytest tests/ -v -s

# 失败时停在第一个失败处
pytest tests/ -v -x
```

## 6. 测试认证接口

```python
# test_auth.py
import pytest
from fastapi.testclient import TestClient
from main import app

client = TestClient(app)

def test_register():
    response = client.post("/register", json={
        "username": "newuser",
        "email": "new@example.com",
        "password": "secure123"
    })
    assert response.status_code == 200
    data = response.json()
    assert data["username"] == "newuser"

def test_login_success():
    """正确密码 → 拿到 Token"""
    # 先注册
    client.post("/register", json={
        "username": "test", "email": "test@x.com", "password": "right"
    })
    # 再登录
    response = client.post("/login", data={
        "username": "test", "password": "right"
    })
    assert response.status_code == 200
    data = response.json()
    assert "access_token" in data
    assert data["token_type"] == "bearer"

def test_login_wrong_password():
    """错误密码 → 401"""
    client.post("/register", json={
        "username": "test2", "email": "t@x.com", "password": "right"
    })
    response = client.post("/login", data={
        "username": "test2", "password": "wrong_password"
    })
    assert response.status_code == 401

def test_protected_route_without_token():
    """未登录访问受保护接口 → 401"""
    response = client.get("/me")
    assert response.status_code == 401

def test_protected_route_with_token():
    """携带 Token 访问受保护接口 → 200"""
    # 注册 + 登录
    client.post("/register", json={
        "username": "tokentest", "email": "tt@x.com", "password": "test"
    })
    login_resp = client.post("/login", data={
        "username": "tokentest", "password": "test"
    })
    token = login_resp.json()["access_token"]

    # 携带 Token 访问
    response = client.get("/me", headers={
        "Authorization": f"Bearer {token}"
    })
    assert response.status_code == 200
    data = response.json()
    assert data["username"] == "tokentest"
```

## 7. pytest 常用技巧

### 7.1 标记（Markers）

```python
import pytest

@pytest.mark.slow       # 标记为慢测试
def test_expensive_operation():
    ...

@pytest.mark.skip(reason="暂不启用")   # 跳过
def test_not_ready():
    ...

@pytest.mark.skipif(sys.version_info < (3, 10), reason="需要 Python 3.10+")
def test_new_feature():
    ...
```

运行标记的测试：

```bash
pytest -m "slow"           # 只跑 slow 标记的
pytest -m "not slow"       # 跳过 slow 标记的
```

### 7.2 参数化测试

同一个测试逻辑，用不同参数跑多次：

```python
import pytest

@pytest.mark.parametrize("username,password,expected_status", [
    ("valid_user", "valid123", 200),          # 正常
    ("",           "valid123", 422),          # 用户名为空
    ("ab",         "valid123", 422),          # 用户名太短
    ("valid_user", "",        422),           # 密码为空
    ("valid_user", "12345",   422),           # 密码太短
])
def test_register_validation(client, username, password, expected_status):
    response = client.post("/register", json={
        "username": username,
        "email": f"{username}@example.com",
        "password": password,
    })
    assert response.status_code == expected_status
```

### 7.3 测试前置与后置

```python
import pytest

@pytest.fixture
def admin_token(client):
    """创建一个 admin 用户，返回其 Token"""
    # 前置：注册 + 登录
    client.post("/register", json={
        "username": "admin", "email": "admin@x.com", "password": "admin123"
    })
    # 手动把角色改成 admin（测试中可直接操作数据库）
    from models import User
    # ... 改角色 ...
    login = client.post("/login", data={"username": "admin", "password": "admin123"})
    return login.json()["access_token"]
    # 后置（如果需要）：yield + 清理代码

def test_admin_endpoint(client, admin_token):
    response = client.get("/admin/users", headers={
        "Authorization": f"Bearer {admin_token}"
    })
    assert response.status_code == 200
```

## 本章小结

```
测试三要素：
  1. TestClient(app)  →  模拟 HTTP 请求，不启动真实服务器
  2. 测试数据库        →  SQLite :memory: 隔离环境，每次测试干净起跑
  3. pytest fixture   →  统一管理测试依赖，conftest.py 全局共享
```

| 做什么 | 怎么测 |
|--------|--------|
| 测 CRUD | POST 创建 → GET 查询验证 → PATCH 更新验证 → DELETE 删除验证 |
| 测验证 | 发送非法数据 → 断言 422 |
| 测认证 | 无 Token → 401，有 Token → 200 |
| 测权限 | 低权限用户访问管理员接口 → 403 |
| 测边界 | 空字符串、超长输入、特殊字符、负数 ID |

> 📖 **下一章：** [5.2-依赖覆盖与Mock.md](5.2-依赖覆盖与Mock.md) — 学会用 `dependency_overrides` 替换外部依赖，让测试不依赖真实服务。

4. 设计一个 `conftest.py`，让所有测试共享 `client` 和 `db` fixture
