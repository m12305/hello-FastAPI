# 6.2 WebSocket

> HTTP 是"你问我才答"——WebSocket 是"打开一条线，双方随时说"——从实时聊天到股价推送，掌握长连接通信。

---

## 1. 为什么需要 WebSocket？

### HTTP 的困境

```
HTTP（请求-响应模式）：
  客户端: 有新消息吗？  →  服务端: 没有
  客户端: 有新消息吗？  →  服务端: 没有
  客户端: 有新消息吗？  →  服务端: 有了！给你。
  客户端: 有新消息吗？  →  服务端: 没有
  ...

问题：
  - 大量无效请求（99% 的轮询都是白问）
  - 延迟高（两条消息之间至少隔一个轮询间隔）
  - 浪费带宽（每个请求都带完整 HTTP 头）
```

### WebSocket 的解决

```
WebSocket（持久连接）：
  客户端 → 服务端: 我们建立 WebSocket 连接！
  服务端 → 客户端: 好的，连接已建立
  ── 连接保持 ──
  服务端 → 客户端: 新消息来了！
  客户端 → 服务端: 我发了一条消息
  服务端 → 客户端: 又一个新消息！
  ...

优势：
  - 服务器可以主动推送（不需要客户端轮询）
  - 延迟极低（毫秒级）
  - 连接建立后只有数据帧（无 HTTP 头开销）
```

| | HTTP | WebSocket |
|------|------|------|
| **通信方向** | 客户端请求 → 服务端响应 | 双向（全双工） |
| **连接** | 每次请求建立新连接（或 Keep-Alive 复用） | 一次握手，持久连接 |
| **谁主动** | 只有客户端能发起 | 客户端和服务器都能主动发 |
| **适用** | REST API、页面浏览、文件下载 | 聊天、通知、实时数据、协作编辑 |
| **协议** | HTTP/1.1, HTTP/2 | ws:// 或 wss://（升级自 HTTP） |

## 2. WebSocket 连接建立过程

```
客户端                        服务端
  │                             │
  │  GET /chat                  │
  │  Upgrade: websocket         │  ← 这是普通 HTTP 请求，但要求"升级"
  │  Connection: Upgrade         │
  │  Sec-WebSocket-Key: xxx     │
  │─────────────────────────────>│
  │                             │
  │  HTTP/1.1 101 Switching     │  ← 服务器同意升级
  │  Upgrade: websocket          │
  │<─────────────────────────────│
  │                             │
  │  ← 现在走 WebSocket 协议了 → │  ← 此后不再是 HTTP！
  │                             │
```

> 💡 关键：WebSocket 通过一条 HTTP 请求"升级"而来，之后就不再是 HTTP 了。

## 3. FastAPI 的 WebSocket 路由

### 3.1 最简单的 WebSocket 端点

```python
from fastapi import FastAPI, WebSocket

app = FastAPI()

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    # ① 首先必须"接受"连接
    await websocket.accept()

    # ② 收发消息
    while True:
        data = await websocket.receive_text()      # 接收
        await websocket.send_text(f"你说: {data}")  # 回复（回声）
```

### 3.2 客户端连接（浏览器 JavaScript）

```javascript
// 在浏览器控制台中运行
const ws = new WebSocket("ws://127.0.0.1:8000/ws");

ws.onopen = () => console.log("已连接");
ws.onmessage = (event) => console.log("收到:", event.data);
ws.send("你好！");  // 发送消息
```

### 3.3 收发不同数据类型

```python
@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()

    while True:
        # 接收消息的三种方式
        text = await websocket.receive_text()           # 接收文本
        # data = await websocket.receive_bytes()        # 接收二进制
        # json = await websocket.receive_json()         # 接收 JSON

        # 发送消息的三种方式
        await websocket.send_text(f"收到文本: {text}")
        # await websocket.send_bytes(b"\x00\x01")       # 发送二进制
        # await websocket.send_json({"msg": text})       # 发送 JSON
```

## 4. 连接管理与心跳检测

### 4.1 处理连接断开

```python
from fastapi import WebSocket, WebSocketDisconnect

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    try:
        while True:
            data = await websocket.receive_text()
            await websocket.send_text(f"收到: {data}")
    except WebSocketDisconnect:
        # 客户端断开连接（关闭标签页、网络断开等）
        print("客户端已断开")
    except Exception as e:
        # 其他异常
        print(f"连接异常: {e}")
```

### 4.2 心跳检测（Ping/Pong）

```python
@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    try:
        while True:
            # 每 30 秒发送一个 ping，确认连接还活着
            await asyncio.sleep(30)
            try:
                await websocket.send_json({"type": "ping"})
            except WebSocketDisconnect:
                break
    except WebSocketDisconnect:
        print("心跳检测到客户端已断开")
```

> 💡 WebSocket 协议自带 ping/pong 帧，Starlette 自动处理。应用层的"心跳"更多是业务需要。

## 5. 简易聊天室

### 5.1 服务端

```python
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from typing import list

app = FastAPI()

class ConnectionManager:
    """管理所有 WebSocket 连接的类"""

    def __init__(self):
        # 存储所有活跃连接
        self.active_connections: list[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        """接受新连接"""
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        """移除连接"""
        self.active_connections.remove(websocket)

    async def broadcast(self, message: str):
        """向所有连接的客户端广播消息"""
        for connection in self.active_connections:
            await connection.send_text(message)

manager = ConnectionManager()


@app.websocket("/ws/chat")
async def chat_endpoint(websocket: WebSocket):
    await manager.connect(websocket)
    try:
        while True:
            data = await websocket.receive_text()
            await manager.broadcast(f"有人说: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast("有人离开了聊天室")
```

```html
            if (e.key === "Enter") send();
```

## 6. 广播与私聊模式

### 6.1 带用户名的广播

```python
class ConnectionManager:
    def __init__(self):
        # 改为字典：websocket → username
        self.active_connections: dict[WebSocket, str] = {}

    async def connect(self, websocket: WebSocket, username: str):
        await websocket.accept()
        self.active_connections[websocket] = username

    def disconnect(self, websocket: WebSocket):
        username = self.active_connections.pop(websocket, "未知")
        return username  # 返回用户名，方便通知"xxx 离开了"

    async def broadcast(self, message: str, sender: str):
        for connection in self.active_connections:
            await connection.send_text(f"[{sender}]: {message}")

    async def send_personal(self, message: str, websocket: WebSocket):
        """发送私密消息给指定连接"""
        await websocket.send_text(message)

    async def send_to_user(self, message: str, username: str):
        """发送消息给指定用户"""
        for ws, name in self.active_connections.items():
            if name == username:
                await ws.send_text(message)
                return True
        return False
```

### 6.2 私聊端点

```python
@app.websocket("/ws/chat/{username}")
async def chat_with_username(websocket: WebSocket, username: str):
    await manager.connect(websocket, username)
    await manager.broadcast(f"{username} 加入了聊天室", "系统")

    try:
        while True:
            # 接收的消息格式: "收件人:消息内容"
            data = await websocket.receive_text()

            if ":" in data:
                target, message = data.split(":", 1)
                # 私聊
                sent = await manager.send_to_user(f"[{username} 悄悄说]: {message}", target)
                if not sent:
                    await manager.send_personal(f"用户 {target} 不在线", websocket)
            else:
                # 广播
                await manager.broadcast(data, username)

    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast(f"{username} 离开了聊天室", "系统")
```

## 7. WebSocket 与后台任务结合

WebSocket 经常和后台任务一起用——后台任务"观察"某些变化，主动推送：

```python
import asyncio

# 模拟一个发布消息的队列
notification_queue: asyncio.Queue = asyncio.Queue()

@app.on_event("startup")
async def start_notification_consumer():
    """启动时创建一个后台任务，不断消费通知"""
    asyncio.create_task(consume_notifications())

async def consume_notifications():
    """后台消费者：有新通知就推送给所有 WebSocket 客户端"""
    while True:
        notification = await notification_queue.get()
        await manager.broadcast(f"📢 通知: {notification}")


@app.websocket("/ws/notifications")
async def notification_endpoint(websocket: WebSocket):
    """订阅通知"""
    await manager.connect(websocket, "anonymous")
    try:
        while True:
            await websocket.receive_text()  # 保持连接
    except WebSocketDisconnect:
        manager.disconnect(websocket)


@app.post("/admin/notify")
async def push_notification(message: str):
    """管理员推送一条通知"""
    await notification_queue.put(message)
    return {"sent": True}
```

## 8. 生产环境的 WebSocket

### 部署注意事项

```nginx
# Nginx 配置 WebSocket 代理
location /ws {
    proxy_pass http://uvicorn;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;        # ← 关键！
    proxy_set_header Connection "upgrade";          # ← 关键！
    proxy_set_header Host $host;
    proxy_read_timeout 86400s;                     # 长连接超时设大
}
```

| 要点 | 说明 |
|------|------|
| **反向代理** | Nginx / Traefik 必须配置 `Upgrade` + `Connection: upgrade` |
| **Gunicorn** | 不能直接跑 WebSocket，要用 `uvicorn` 或 `daphne` |
| **伸缩** | 多实例时需要用 Redis Pub/Sub 同步消息（一个实例发了，其他实例也能推） |
| **认证** | WebSocket 不能发 `Authorization` 头，通常通过 URL 参数传 Token：`ws://host/chat?token=xxx` |

### WebSocket 认证示例

```python
@app.websocket("/ws/chat")
async def chat_auth(websocket: WebSocket, token: str = None):

        return
```

## 本章小结

```
WebSocket 全景：

  客户端                        服务端
    │                            │
    ├── GET /ws/chat ───────────>│  HTTP Upgrade 握手
    │<── 101 Switching Protocols │
    │                            │
    ├── 消息1 ──────────────────>│
    │<── 推送A ─────────────────│  ← 服务器主动推
    ├── 消息2 ──────────────────>│
    │<── 推送B ─────────────────│
    │                            │
    ├── 断开 ──────────────────>│  WebSocketDisconnect
    │                            │
```

| 层级 | 实现方式 |
|------|---------|
| **基础** | `@app.websocket("/ws")` + `accept()` + `receive_text/send_text()` |
| **异常** | `try/except WebSocketDisconnect` |
| **广播** | `ConnectionManager` 维护连接列表，遍历 `send_text()` |
| **私聊** | `ConnectionManager` 维护 `{websocket: username}` 映射 |
| **心跳** | `asyncio.sleep(30)` + `send_json({"type": "ping"})` |
| **认证** | URL 参数传 Token：`ws://host/chat?token=xxx` |
