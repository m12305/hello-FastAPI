# 6.3 事件钩子与生命周期

> 应用启动时初始化数据库连接池，关闭时释放资源——掌握 FastAPI 的生命周期管理，让应用体面地"生"和优雅地"死"。

---

## 1. FastAPI 应用的三个生命阶段

```
时间线 ────────────────────────────────────────────────────>

  启动阶段             运行阶段                 关闭阶段
  ┌────────┐    ┌────────────────────┐    ┌────────┐
  │ 初始化  │    │     处理请求        │    │ 清理   │
  │ 资源    │ →  │   (正常业务)        │ →  │释放    │
  │ 连接    │    │                    │    │资源    │
  └────────┘    └────────────────────┘    └────────┘

  startup        lifespan / 请求处理       shutdown
  事件           生命周期中               事件
```

> 💡 关键理解：FastAPI 和 Starlette 本身是无状态的，但你的应用通常需要状态（数据库连接池、Redis 连接、模型缓存）。生命周期管理就是帮你正确地初始化和清理这些状态。

---

## 2. 旧方式：`@app.on_event()`

这是你之前已经在用的方式（阶段 3/4/5 的 demo 都在用）：

```python
from fastapi import FastAPI

app = FastAPI()

@app.on_event("startup")
async def startup():
    """应用启动时执行 —— 建表、建立连接等"""
    Base.metadata.create_all(bind=engine)
    print("✅ 数据库表已创建")
    # 其他初始化：连接 Redis、加载模型等

@app.on_event("shutdown")
async def shutdown():
    """应用关闭时执行 —— 释放连接、关闭资源等"""
    engine.dispose()
    print("👋 数据库连接已释放")
```

| 事件 | 触发时机 | 典型用途 |
|------|---------|---------|
| `startup` | 第一条请求到达前 | 建表、创建连接池、加载配置、预热缓存 |
| `shutdown` | 应用停止后 | 关闭连接池、释放资源、刷新缓冲区 |

### 问题

```python
# ❌ on_event 的问题：
# 1. startup 和 shutdown 之间没有共享状态的方式
# 2. 多个 startup/shutdown 事件之间的执行顺序不保证
# 3. 无法优雅地传递"启动时创建的"对象给请求处理函数
```

---

## 3. 新方式：`lifespan` 上下文管理器（推荐）

FastAPI/Starlette 推荐用 `lifespan` 替代 `on_event`：

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI

# ═══════════════════════════════════════════════════════════
# 定义 lifespan 上下文管理器
# ═══════════════════════════════════════════════════════════

@asynccontextmanager
async def lifespan(app: FastAPI):
    """
    FastAPI 应用的生命周期管理器。

    yield 之前 = startup 逻辑
    yield 之后 = shutdown 逻辑
    """
    # ═══ 启动阶段 ═══
    print("🚀 应用启动中...")
    Base.metadata.create_all(bind=engine)
    print("✅ 数据库表已创建")

    # ═══ 把控制权交给应用 ═══
    yield  # ← 应用在这里"存活"、处理请求

    # ═══ 关闭阶段 ═══
    print("👋 应用正在关闭...")
    engine.dispose()
    print("✅ 数据库连接已释放")


# ═══════════════════════════════════════════════════════════
# 创建应用时传入 lifespan
# ═══════════════════════════════════════════════════════════

app = FastAPI(lifespan=lifespan)
```

### 为什么 lifespan 更好？

```
lifespan 的结构：
  @asynccontextmanager        ← Python 标准库
  async def lifespan(app):
      # 所有初始化代码（替代 on_event("startup")）
      init_db()
      init_redis()
      yield  # ← 清晰的分界线
      # 所有清理代码（替代 on_event("shutdown")）
      close_redis()
      close_db()
```

| 对比 | `@app.on_event()` | `lifespan` |
|------|-------------------|-----------|
| **状态共享** | 困难（需要全局变量） | 通过 yield 传递 |
| **启动完毕确认** | 无显式确认 | yield 即确认"就绪" |
| **异常处理** | startup 抛异常直接崩 | 可在外层 try/catch |
| **官方推荐** | 遗留方式 | ✅ 当前推荐 |
| **代码组织** | 分散在多处 | 集中在同一个函数 |

## 4. 在 lifespan 中管理共享资源

### 4.1 管理数据库连接池 + Redis 客户端

```python
import redis.asyncio as redis
from contextlib import asynccontextmanager
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

# ── 全局变量保存"启动时创建"的对象 ──
redis_client: redis.Redis | None = None
async_session_factory: async_sessionmaker | None = None


@asynccontextmanager
async def lifespan(app: FastAPI):
    global redis_client, async_session_factory

    # ═══ 启动阶段 ═══
    # 创建异步数据库引擎
    engine = create_async_engine(
        "mysql+asyncmy://root:password@localhost:3306/mydb",
        pool_size=20,
        max_overflow=40,
        pool_pre_ping=True,        # 连接前检查有效性
        pool_recycle=3600,         # 每小时回收连接
    )
    async_session_factory = async_sessionmaker(engine, class_=AsyncSession)

    # 创建 Redis 连接池
    redis_client = redis.from_url("redis://localhost:6379/0")

    print("✅ 数据库连接池 + Redis 已就绪")
    yield  # ← 应用存活

    # ═══ 关闭阶段 ═══
    if redis_client:
        await redis_client.close()
    if engine:
        await engine.dispose()
    print("👋 所有连接已释放")


app = FastAPI(lifespan=lifespan)
```

### 4.2 将资源注入到请求中

```python
def get_redis() -> redis.Redis:
    """依赖注入：获取 Redis 客户端"""
    if redis_client is None:
        raise RuntimeError("Redis 客户端未初始化（lifespan 阶段可能失败）")
    return redis_client


@app.get("/cache/{key}")
async def get_cache(key: str, r: redis.Redis = Depends(get_redis)):
    value = await r.get(key)
    if value is None:
        raise HTTPException(404, "缓存未命中")
    return {"key": key, "value": value.decode()}
```

### 4.3 更优雅的方式：把状态挂在 app.state 上

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 把状态存在 app.state 上 —— 不需要全局变量！
    app.state.db_engine = create_async_engine(...)
    app.state.redis = redis.from_url(...)
    app.state.model = load_ml_model()  # 启动时加载一次 ML 模型
    app.state.config = load_config()

    yield  # ← 应用运行中，所有端点都能通过 app.state 访问这些资源

    await app.state.redis.close()
    await app.state.db_engine.dispose()


# 在依赖中访问
def get_redis(request: Request) -> redis.Redis:
    """从 app.state 中获取 Redis"""
    return request.app.state.redis


@app.get("/info")
def info(request: Request):
    return {
        "db_ready": request.app.state.db_engine is not None,
        "redis_ready": request.app.state.redis is not None,
    }
```

## 5. 优雅关闭

### 5.1 等待正在进行的请求完成

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 初始化资源...
    yield

    # 优雅关闭：先标记"不再接收新请求"，再等现有请求完成
    print("🛑 正在优雅关闭...")
    # FastAPI/Starlette 默认会等待正在处理的请求完成
    # 但你可以在这里加上额外的等待逻辑
    await asyncio.sleep(2)  # 给任务一些收尾时间
    print("✅ 所有请求已处理完毕")
```

### 5.2 取消后台任务

```python
import asyncio

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动一个后台任务
    background_task = asyncio.create_task(periodic_cleanup())

    yield  # ← 应用存活期间，后台任务一直在跑

    # 关闭时取消后台任务
    background_task.cancel()
    try:
        await background_task
    except asyncio.CancelledError:
        print("✅ 后台任务已取消")


async def periodic_cleanup():
    """每 60 秒清理过期数据"""
    while True:
        try:
            # 清理逻辑...
            print("🧹 清理完成")
        except Exception as e:
            print(f"清理出错: {e}")
        await asyncio.sleep(60)
```

## 6. 完整示例

将前面所学整合到一个完整应用中：

```python
from contextlib import asynccontextmanager
import redis.asyncio as redis
from fastapi import FastAPI, Request, Depends, HTTPException
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker, AsyncSession

# ═══════════════════════════════════════════════════════════
# lifespan 管理所有资源
# ═══════════════════════════════════════════════════════════

@asynccontextmanager
async def lifespan(app: FastAPI):
    # ── 启动阶段 ──
    app.state.db_engine = create_async_engine(
        "mysql+asyncmy://root:password@localhost:3306/mydb",
        pool_size=20, pool_pre_ping=True,
    )
    app.state.async_session = async_sessionmaker(
        app.state.db_engine, class_=AsyncSession,
    )
    app.state.redis = redis.from_url("redis://localhost:6379/0")

    # 健康检查：验证连接可用
    try:
        async with app.state.db_engine.connect() as conn:
            await conn.execute(text("SELECT 1"))
        await app.state.redis.ping()
        print("✅ 数据库 + Redis 连接正常")
    except Exception as e:
        print(f"⚠️ 启动失败: {e}")
        raise  # 启动失败就不要继续了

    yield

    # ── 关闭阶段 ──
    await app.state.redis.close()
    await app.state.db_engine.dispose()
    print("👋 资源已释放")


app = FastAPI(lifespan=lifespan)


# ═══════════════════════════════════════════════════════════
# 依赖注入：从 app.state 获取资源
# ═══════════════════════════════════════════════════════════

async def get_db(request: Request):
    async with request.app.state.async_session() as session:
        try:
            yield session
        finally:
            await session.close()

def get_redis(request: Request):
    return request.app.state.redis


# ═══════════════════════════════════════════════════════════
# 端点
# ═══════════════════════════════════════════════════════════

@app.get("/health")
async def health(request: Request):
    """就绪检查 —— K8s / 负载均衡用"""
    try:
        await request.app.state.redis.ping()
        return {"status": "healthy", "redis": "ok"}
    except Exception:
        return JSONResponse(
            {"status": "unhealthy"},
            status_code=503,
        )


@app.get("/cache/{key}")
async def get_cache(key: str, r: redis.Redis = Depends(get_redis)):
    """使用 Redis 读取缓存"""
    val = await r.get(key)
    if val is None:
        raise HTTPException(404, "Key 不存在")
    return {"key": key, "value": val.decode()}
```

## 本章小结

```
FastAPI 生命周期：

  @asynccontextmanager
  async def lifespan(app):
      ┌─────────────────────────┐
      │  初始化阶段               │
      │  • 创建 DB 引擎          │
      │  • 创建 Redis 连接        │
      │  • 加载 ML 模型           │
      │  • 验证连接可用性          │
      ├─────────────────────────┤
      │  yield  ← 应用存活      │
      │        处理请求          │
      ├─────────────────────────┤
      │  清理阶段                 │
      │  • 关闭 Redis            │
      │  • 释放 DB 连接池         │
      │  • 取消后台任务            │
      └─────────────────────────┘
  ```

| 要点 | 说明 |
|------|------|
| **旧方式** | `@app.on_event("startup")` / `@app.on_event("shutdown")` —— 仍可用 |
| **新方式** | `lifespan` 上下文管理器 —— 官方推荐 |
| **状态管理** | 存在 `app.state` 上，通过 `request.app.state` 访问 |
| **优雅关闭** | 取消后台任务、等待请求完成、释放连接 |
| **健康检查** | 在 startup 阶段验证所有外部服务可达 |
