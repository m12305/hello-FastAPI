# 7.2 配置管理

> "把密码写代码里？那是 10 年前的做法。"——用 pydantic-settings 实现类型安全的多环境配置，遵循 12-Factor App 原则。

---

## 1. 为什么需要配置管理？

### 硬编码的灾难

```python
# ❌ 绝对不要这样做！
DATABASE_URL = "mysql+pymysql://root:admin123@localhost:3306/mydb"
SECRET_KEY = "my-secret-key-abc123"
REDIS_URL = "redis://localhost:6379/0"
#               ↑
# 问题：
#   1. 密码泄露到 Git 历史中
#   2. 开发/测试/生产切环境要改代码
#   3. 不同开发者的数据库密码不同 → 冲突
#   4. 服务器上不能改配置 → 每次都要重新部署
```

### 正确的做法

```bash
# .env 文件（不提交到 Git！）
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/mydb
SECRET_KEY=production-secret-key-min-32-chars-long!
REDIS_URL=redis://localhost:6379/0
DEBUG=false
```

```python
# core/config.py（可提交到 Git）
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    DATABASE_URL: str
    SECRET_KEY: str
    REDIS_URL: str = "redis://localhost:6379/0"  # 默认值
    DEBUG: bool = False

    class Config:
        env_file = ".env"  # 自动读取 .env 文件

settings = Settings()  # 全局单例

# 使用：
print(settings.DATABASE_URL)  # 从 .env 中读取，类型自动推断
```

## 2. pydantic-settings 基础

### 2.1 安装

```bash
pip install pydantic-settings
```

### 2.2 基本用法

```python
from pydantic_settings import BaseSettings
from pydantic import Field, field_validator

class Settings(BaseSettings):
    # ── 必填配置（无默认值 → 找不到就报错）──
    DATABASE_URL: str = Field(description="数据库连接 URL")
    SECRET_KEY: str = Field(min_length=32, description="JWT 签名密钥")

    # ── 可选配置（有默认值）──
    APP_NAME: str = "My API"
    APP_VERSION: str = "1.0.0"
    DEBUG: bool = False
    PORT: int = Field(default=8000, ge=1, le=65535)

    # ── 复杂类型 ──
    ALLOWED_HOSTS: list[str] = ["localhost", "127.0.0.1"]

    # ── 自定义校验 ──
    @field_validator("DATABASE_URL")
    @classmethod
    def validate_db_url(cls, v: str) -> str:
        if not v.startswith(("mysql://", "mysql+pymysql://", "sqlite://", "postgresql://")):
            raise ValueError(f"不支持的数据库类型: {v}")
        return v

    class Config:
        env_file = ".env"          # 读取 .env 文件
        env_file_encoding = "utf-8"
        case_sensitive = True       # 环境变量区分大小写
        extra = "forbid"            # 禁止未定义的配置项
```

### 2.3 加载顺序

pydantic-settings 按以下顺序查找配置（后面的覆盖前面的）：

```
1. 默认值（Field(default=...)）
2. .env 文件
3. 系统环境变量（最高优先级）
```

```
示例：DATABASE_URL 的值最终是什么？

  Field(default="sqlite:///dev.db")    → sqlite:///dev.db     ← 默认值
  .env 文件: DATABASE_URL=mysql://...  → mysql://...          ← 覆盖默认值
  export DATABASE_URL=postgresql://... → postgresql://...     ← 最终值（最高优先级）
```

## 3. 多环境配置

### 3.1 目录结构

```
app/core/
├── __init__.py
├── config.py           # 基础配置 + 配置工厂
├── config_dev.py       # 开发环境
├── config_test.py      # 测试环境
└── config_prod.py      # 生产环境
```

```
.env.dev   # 开发环境
.env.test  # 测试环境
.env.prod  # 生产环境
```

### 3.2 代码实现

```python
# app/core/config.py
from pydantic_settings import BaseSettings
from pydantic import Field
import os

class Settings(BaseSettings):
    """基础配置 —— 所有环境共享"""

    # ── 应用 ──
    APP_NAME: str = "My API"
    APP_VERSION: str = "1.0.0"
    ENVIRONMENT: str = Field(default="development")

    # ── 数据库 ──
    DATABASE_URL: str
    DB_POOL_SIZE: int = 20
    DB_POOL_RECYCLE: int = 3600
    DB_ECHO: bool = False  # SQL 日志

    # ── JWT ──
    SECRET_KEY: str = Field(min_length=32)
    ACCESS_TOKEN_EXPIRE_MINUTES: int = 30

    # ── Redis ──
    REDIS_URL: str = "redis://localhost:6379/0"

    # ── 日志 ──
    LOG_LEVEL: str = "INFO"

    # ── CORS ──
    CORS_ORIGINS: list[str] = ["http://localhost:3000"]

    @property
    def is_dev(self) -> bool:
        return self.ENVIRONMENT == "development"

    @property
    def is_prod(self) -> bool:
        return self.ENVIRONMENT == "production"

    @property
    def is_test(self) -> bool:
        return self.ENVIRONMENT == "test"


class DevSettings(Settings):
    """开发环境配置"""
    DB_ECHO: bool = True     # 开发环境看 SQL 日志
    LOG_LEVEL: str = "DEBUG"


class TestSettings(Settings):
    """测试环境配置"""
    DATABASE_URL: str = "sqlite:///:memory:"   # 测试用内存库
    DB_ECHO: bool = False
    LOG_LEVEL: str = "WARNING"


class ProdSettings(Settings):
    """生产环境配置"""
    DB_ECHO: bool = False    # 生产环境绝不能输出 SQL！
    LOG_LEVEL: str = "WARNING"
    DEBUG: bool = False


# ── 配置工厂 ──
def get_settings() -> Settings:
    """根据环境变量返回对应的配置实例"""
    env = os.getenv("ENVIRONMENT", "development")
    env_file_map = {
        "development": ".env.dev",
        "test": ".env.test",
        "production": ".env.prod",
    }
    env_file = env_file_map.get(env, ".env.dev")

    settings_class_map = {
        "development": DevSettings,
        "test": TestSettings,
        "production": ProdSettings,
    }
    settings_class = settings_class_map.get(env, DevSettings)

    return settings_class(_env_file=env_file)


# 全局单例
settings = get_settings()
```

## 4. 配置的最佳实践

### 4.1 .gitignore 配置

```bash
# .gitignore
.env          # 不要提交任何 .env 文件
.env.*        # 包括多环境配置文件
*.local       # 本地配置

# 但要提供一个模板文件
# .env.example（可提交）
```

```bash
# .env.example —— 提交到 Git，告诉其他开发者需要哪些配置
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/mydb
SECRET_KEY=change-me-to-a-random-string
REDIS_URL=redis://localhost:6379/0
ENVIRONMENT=development
```

### 4.2 在 Docker 中注入配置

```yaml
# docker-compose.yml
services:
  api:
    build: .
    environment:
      - DATABASE_URL=mysql+pymysql://root:${DB_PASSWORD}@db:3306/mydb
      - SECRET_KEY=${SECRET_KEY}
      - ENVIRONMENT=production
    env_file:
      - .env.prod  # 也可以直接挂文件
```

### 4.3 敏感配置使用 Secret 管理

```python
# 生产环境：从密钥管理服务获取
# Kubernetes Secret / AWS Secrets Manager / HashiCorp Vault

import os

class ProdSettings(Settings):
    # 优先从环境变量读（K8s Secret 注入），其次从文件读
    SECRET_KEY: str = Field(default="")

    @field_validator("SECRET_KEY")
    @classmethod
    def load_secret(cls, v: str) -> str:
        if v:
            return v
        # 尝试从 Docker Secret / K8s Secret 文件读取
        secret_file = os.getenv("SECRET_KEY_FILE", "/run/secrets/secret_key")
        try:
            with open(secret_file) as f:
                return f.read().strip()
        except FileNotFoundError:
            raise ValueError("SECRET_KEY 未找到！请设置环境变量或挂载 Secret 文件")
```

## 5. 12-Factor App 配置原则

| 原则 | 说明 | FastAPI 实现 |
|------|------|-------------|
| **代码与配置分离** | 配置不写死在代码里 | `.env` 文件 |
| **环境变量存储** | 通过环境变量注入配置 | `pydantic-settings` 自动读取 |
| **按环境区分** | 开发/测试/生产独立配置 | `config_dev.py` / `config_prod.py` |
| **版本控制隔离** | 不提交密钥到 Git | `.env` 在 `.gitignore` 中 |
| **一次性部署** | 通过改配置部署，不通过改代码 | 改环境变量，不改代码 |

## 本章小结

```
配置管理的三个层次：

  1. pydantic-settings
     └─ 类型安全、自动校验、.env 文件读取

  2. 多环境策略
     └─ development / test / production

  3. Secret 管理
     └─ K8s Secret / Vault / AWS Secrets Manager
```

| 要点 | 最佳实践 |
|------|---------|
| **安装** | `pip install pydantic-settings` |
| **基类** | `class Settings(BaseSettings)` |
| **校验** | `Field(min_length=...)` + `@field_validator` |
| **文件** | `.env` 放项目根目录，自动加载 |
| **多环境** | 工厂模式 `get_settings()` 按环境返回不同实例 |
| **Git** | `.env` 加入 `.gitignore`，提交 `.env.example` 模板 |
