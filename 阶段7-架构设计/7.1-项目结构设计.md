# 7.1 项目结构设计

> 从"所有代码塞一个文件"到"分层清晰的大型项目"——学会 FastAPI 工程化的目录结构设计，让代码可维护、可扩展、可交接。

---

## 1. 为什么需要规范的项目结构？

### 一个文件的噩梦

```
# ❌ 所有代码在一个 main.py 中（5000 行+）
main.py:
  ├── 数据库模型（500 行）
  ├── Pydantic 模型（300 行）
  ├── 认证逻辑（400 行）
  ├── 用户 CRUD（600 行）
  ├── 文章 CRUD（600 行）
  ├── 管理员端点（400 行）
  ├── 文件上传（300 行）
  ├── 邮件发送（200 行）
  └── ...（越来越长）
```

| 问题 | 后果 |
|------|------|
| 找不到代码 | 改一个功能要在 5000 行里翻 |
| 改一处炸一片 | 都耦合在一起，不敢动 |
| 多人协作冲突 | 所有人都在改同一个文件 |
| 无法复用 | 认证逻辑被埋在主文件里，新项目没法用 |
| 测试困难 | 所有依赖搅在一起，Mock 不了 |

### 分层后的清晰结构

```
👤 新同事: "用户注册的校验逻辑在哪？"
👨‍💻 你: "看 app/services/user_service.py，第 42 行"
           ↑ 精确到行，5 秒定位
```

## 2. 小型项目 vs 大型项目的目录结构

### 2.1 小型项目（< 10 个端点）

```
myapp/
├── main.py           # 应用入口（所有路由写这里）
├── models.py         # ORM + Pydantic 模型
├── database.py       # 引擎 + 会话 + get_db
├── auth.py           # JWT 认证 + 权限检查
├── schemas.py        # 请求/响应模型
└── requirements.txt
```

适用：个人项目、微服务、快速原型。

### 2.2 中型项目（10-50 个端点）

```
myapp/
├── app/
│   ├── main.py            # 应用入口（注册路由 + 事件）
│   ├── database.py        # 引擎 + 会话
│   ├── models.py          # ORM 模型
│   ├── schemas.py         # Pydantic 模型
│   ├── auth.py            # 认证核心
│   └── routers/
│       ├── __init__.py
│       ├── users.py       # /users 相关路由
│       ├── posts.py       # /posts 相关路由
│       └── admin.py       # /admin 相关路由
├── tests/
└── requirements.txt
```

适用：小团队项目、阶段 5 demo 用的就是这种。

### 2.3 大型项目（50+ 端点，推荐）

```
project/
├── app/
│   ├── main.py              # 🚪 应用入口（仅注册路由 + 挂载中间件 + lifespan）
│   │
│   ├── api/                 # 🌐 路由层（只负责"接请求→调服务→返响应"）
│   │   ├── __init__.py
│   │   ├── v1/              #   版本 1
│   │   │   ├── __init__.py
│   │   │   ├── router.py    #   汇总所有 v1 路由
│   │   │   └── endpoints/
│   │   │       ├── users.py
│   │   │       ├── posts.py
│   │   │       └── auth.py
│   │   └── v2/              #   版本 2（如果存在）
│   │
│   ├── core/                # ⚙️ 核心配置与安全
│   │   ├── __init__.py
│   │   ├── config.py        #   配置管理（pydantic-settings）
│   │   ├── security.py      #   JWT + 密码哈希 + 权限依赖
│   │   ├── dependencies.py  #   通用依赖（get_db, get_redis, get_current_user）
│   │   └── exceptions.py    #   自定义异常 + 异常处理器
│   │
│   ├── models/              # 🗄️ 数据库 ORM 模型（只定义表结构，不含业务逻辑）
│   │   ├── __init__.py
│   │   ├── base.py          #   Base 基类
│   │   ├── user.py          #   User 表
│   │   └── post.py          #   Post 表
│   │
│   ├── schemas/             # 📋 Pydantic 数据校验模型（请求/响应）
│   │   ├── __init__.py
│   │   ├── user.py          #   UserCreate, UserResponse, UserUpdate
│   │   └── post.py          #   PostCreate, PostResponse
│   │
│   ├── services/            # 🧠 业务逻辑层（核心——所有复杂逻辑放这里）
│   │   ├── __init__.py
│   │   ├── user_service.py  #   register, login, update_profile
│   │   └── post_service.py  #   create_post, publish, archive
│   │
│   ├── crud/                # 🗃️ 数据库操作层（纯 CRUD，不含业务校验）
│   │   ├── __init__.py
│   │   ├── base.py          #   通用 CRUD 基类
│   │   ├── user_crud.py     #   User 的 CRUD
│   │   └── post_crud.py     #   Post 的 CRUD
│   │
│   ├── db/                  # 💾 数据库基础设施
│   │   ├── __init__.py
│   │   ├── session.py       #   引擎 + SessionLocal + get_db
│   │   └── base.py          #   DeclarativeBase
│   │
│   ├── utils/               # 🛠 工具函数（纯函数，无状态）
│   │   ├── __init__.py
│   │   ├── pagination.py    #   分页工具
│   │   └── hashing.py       #   哈希工具
│   │
│   └── middleware/           # 🔗 中间件
│       ├── __init__.py
│       ├── logging.py       #   日志中间件
│       └── cors.py          #   CORS 配置
│
├── tests/                   # 🧪 测试（结构镜像 app/）
│   ├── conftest.py
│   ├── api/
│   ├── services/
│   └── crud/
│
├── alembic/                 # 📦 数据库迁移
│   ├── versions/
│   └── env.py
│
├── .github/workflows/       # CI/CD
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
└── .env
```

## 3. 各层的职责与边界

```
请求                   路由层                    服务层                    CRUD 层               数据库
                      (api/v1/)                (services/)              (crud/)
                        │                         │                        │
POST /register ────> validate  ────────────> register()  ────────────> create_user()  ────> INSERT
                        │ 输入                    │ 业务                    │ SQL
                        │                         │                        │
                    只做数据校验：          处理业务逻辑：            只做数据库操作：
                    - 字段非空              - 用户名唯一性            - 插入行
                    - 类型正确              - 密码哈希                - 查询行
                    - 格式合法              - 发送欢迎邮件             - 更新行
                                            - 写审计日志
```

### 各层原则

```python
# ── api/endpoints/users.py（路由层）──
# 职责：接请求 → 调服务 → 返响应
# 不超过 15 行！

@router.post("/register", response_model=UserResponse, status_code=201)
def register(
    data: UserCreate,
    db: Session = Depends(get_db),
    background_tasks: BackgroundTasks = BackgroundTasks(),
):
    """用户注册。只做数据校验和路由，业务逻辑委托给 service"""
    from app.services.user_service import UserService
    user = UserService(db).register(data, background_tasks)
    return user


# ── services/user_service.py（服务层）──
# 职责：编排业务逻辑
# 可以有 50-100 行 —— 业务逻辑就该详细！

class UserService:
    def __init__(self, db: Session):
        self.db = db

    def register(self, data: UserCreate, bg: BackgroundTasks) -> User:
        # 1. 检查用户名唯一性
        existing = self.db.query(User).filter(User.username == data.username).first()
        if existing:
            raise DuplicateUserException(data.username)

        # 2. 密码哈希
        hashed = get_password_hash(data.password)

        # 3. 创建用户
        from app.crud.user_crud import UserCRUD
        user = UserCRUD(self.db).create(
            username=data.username,
            email=data.email,
            hashed_password=hashed,
        )

        # 4. 后台任务
        bg.add_task(send_welcome_email, user.email, user.username)
        bg.add_task(write_audit_log, "register", user.id)

        return user


# ── crud/user_crud.py（CRUD 层）──
# 职责：只做数据库操作，不含任何业务规则

class UserCRUD:
    def __init__(self, db: Session):
        self.db = db

    def create(self, **kwargs) -> User:
        user = User(**kwargs)
        self.db.add(user)
        self.db.commit()
        self.db.refresh(user)
        return user

    def get_by_id(self, user_id: int) -> User | None:
        return self.db.get(User, user_id)

    def get_by_username(self, username: str) -> User | None:
        return self.db.query(User).filter(User.username == username).first()
```

## 4. APIRouter：模块化路由

```python
# api/v1/router.py — 汇总所有 v1 路由
from fastapi import APIRouter
from app.api.v1.endpoints import users, posts, auth

router = APIRouter()
router.include_router(auth.router, prefix="/auth", tags=["认证"])
router.include_router(users.router, prefix="/users", tags=["用户"])
router.include_router(posts.router, prefix="/posts", tags=["文章"])

# 最终路径：
# /api/v1/auth/register
# /api/v1/auth/login
# /api/v1/users
# /api/v1/users/{id}
# /api/v1/posts
# /api/v1/posts/{id}
```

```python
# main.py — 入口文件，极其简洁
from fastapi import FastAPI
from app.api.v1.router import router as v1_router
from app.core.config import settings

app = FastAPI(title=settings.APP_NAME, version=settings.APP_VERSION)

app.include_router(v1_router, prefix="/api/v1")

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("app.main:app", reload=True)
```

## 5. 重构路线图：从单文件到大项目

```
第 1 步 — 把路由从 main.py 抽到 routers/（30 分钟）
  main.py: 路由定义 → routers/users.py + routers/posts.py

第 2 步 — 分离 ORM 和 Pydantic 模型（15 分钟）
  一个 models.py → models/（ORM） + schemas/（Pydantic）

第 3 步 — 抽离配置（20 分钟）
  环境变量、密钥 → core/config.py（pydantic-settings）

第 4 步 — 抽离依赖（20 分钟）
  get_db、get_current_user → core/dependencies.py

第 5 步 — 抽离业务逻辑（1 小时）
  端点中的业务逻辑 → services/

第 6 步 — 抽离数据库操作（30 分钟）
  直接写在服务中的查询 → crud/

不需要一步到位！按需拆分，跟着项目复杂度走。
```

## 本章小结

```
项目结构演进路线：

  小型项目             中型项目               大型项目
  (1-10 端点)        (10-50 端点)          (50+ 端点)

  main.py        app/                 app/
  models.py      ├── main.py          ├── api/       (路由层)
  database.py    ├── database.py      ├── core/      (配置+安全)
  auth.py        ├── models.py        ├── services/  (业务逻辑)
  schemas.py     ├── schemas.py       ├── crud/      (数据库操作)
                 ├── auth.py          ├── models/    (ORM 模型)
                 └── routers/         ├── schemas/   (Pydantic 模型)
                     ├── users.py     ├── db/        (数据库基础设施)
                     ├── posts.py     ├── utils/     (工具函数)
                     └── admin.py     └── middleware/ (中间件)
```

| 原则 | 说明 |
|------|------|
| **路由层要薄** | 只做校验+路由，不写业务逻辑 |
| **服务层要厚** | 业务逻辑全放 services/，可独立测试 |
| **CRUD 要纯** | 只做数据库操作，不含任何业务规则 |
| **横切关注点统一** | 认证、日志、异常处理放 core/ 或 middleware/ |
