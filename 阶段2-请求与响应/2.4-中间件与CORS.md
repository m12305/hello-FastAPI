# 2.4 中间件与 CORS

> 中间件让你在请求到达路由**之前**和响应离开**之后**插入自定义逻辑——它是全局横切关注点的最佳处理方式。

---

## 1. 中间件是什么？

### 一个比喻

想象一个快递分拣中心。每个包裹（HTTP 请求）从进入仓库到最终送达，要经过多个处理站：
- 第 1 站：安检扫描（安全检测）
- 第 2 站：打标签（记录快递单号）  
- 第 3 站：分类分拣（按目的地路由）
- 然后送到具体的处理窗口（路由函数）
- 返回时反方向再过一遍（后处理）

每经过一站，包裹可以被检查、修改、甚至拒收。这些"站"就是**中间件**。

### 技术视角：洋葱模型

中间件**包裹**了整个路由层，请求穿过每一层，到达核心（路由函数），然后响应再反向穿回来：

```
                      ┌──────────────────────────────┐
                      │         FastAPI 应用           │
                      │                               │
HTTP 请求 ──→ 中间件A ──→ 中间件B ──→ 路由处理函数 ──→ 中间件B ──→ 中间件A ──→ HTTP 响应
              │          │          │                │          │
              ├ 前置处理  │          │                │ 后置处理  │
              └ 修改请求  │          │                └ 修改响应  │
```

你可以这样理解：中间件的**前置部分**是一层层进门（请求），**后置部分**是一层层出门（响应）。进门顺序 = A → B → 路由；出门顺序 = 路由 → B → A。

### 中间件 vs 依赖注入 vs 异常处理器

这三个概念容易混淆，先明确它们的分工：

| 机制 | 作用范围 | 能做什么 | 不能做什么 |
|------|---------|---------|-----------|
| **中间件** | 所有请求（全局） | 修改请求/响应、记录日志、CORS | 精细控制单个端点 |
| **依赖注入** | 指定端点或路由组 | 认证、分页、数据库会话 | 全局生效需手动加到每个端点 |
| **异常处理器** | 全局异常 | 统一错误格式 | 正常请求处理 |

> 简单记忆：要全局生效用中间件，要精细控制用依赖注入，要处理异常用异常处理器。

## 2. 编写自定义中间件

### 2.1 基础模板

```python
from fastapi import FastAPI, Request
import time

app = FastAPI()


@app.middleware("http")
async def add_process_time_header(request: Request, call_next):
    """
    最简单的中间件：
    1. 记录请求到达时间
    2. 调用下一个处理器（call_next）
    3. 记录响应返回时间，添加自定义响应头
    """
    start_time = time.perf_counter()

    # ─── 请求处理阶段（前置）───
    # 可以在这里检查请求、修改请求头、记录日志等

    response = await call_next(request)   # ← 调用下一层（可能是另一个中间件或路由）

    # ─── 响应处理阶段（后置）───
    # 可以在这里修改响应、添加响应头、记录耗时等

    process_time = time.perf_counter() - start_time
    response.headers["X-Process-Time"] = f"{process_time:.4f}"  # 需暴露给 Starlette
    return response
```

> ⚠️ **关键约定**：自定义中间件的 `X-` 前缀响应头默认**不会**暴露给客户端。如果前端需要读取，需在 CORS 配置中添加 `expose_headers=["X-Process-Time"]`。

### 2.2 必须 `await call_next(request)`

```python
# ❌ 错误：忘记 await
@app.middleware("http")
async def broken_middleware(request, call_next):
    response = call_next(request)  # 没有 await → 返回 coroutine 对象，不是 Response
    return response

# ✅ 正确
@app.middleware("http")
async def correct_middleware(request, call_next):
    response = await call_next(request)
    return response
```

### 2.3 可阻塞的中间件

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse


@app.middleware("http")
async def block_bad_user_agents(request: Request, call_next):
    """拦截恶意 User-Agent"""
    user_agent = request.headers.get("User-Agent", "").lower()

    blocked = ["scrapy", "nmap", "sqlmap", "nikto", "curl"]
    # 注意：生产环境不要真的拦截 curl，这里只是演示

    if any(bad in user_agent for bad in blocked):
        # 直接返回，不调用 call_next → 请求永远不会到达路由
        return JSONResponse(
            status_code=403,
            content={"detail": "Forbidden: suspicious user agent"},
        )

    # 正常请求继续
    return await call_next(request)
```

## 3. 实用中间件示例

### 3.1 请求 ID 注入

```python
import uuid
import logging
from fastapi import FastAPI, Request

logger = logging.getLogger("api")


@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    """为每个请求生成唯一 ID，方便日志追踪"""

    # 优先使用客户端传的 X-Request-ID，否则自动生成
    request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))

    # 把 request_id 注入到 request.state，路由中可以通过 request.state.request_id 访问
    request.state.request_id = request_id

    # 注入到日志上下文（配合 logging 的 extra）
    logger.info(f"[{request_id}] {request.method} {request.url.path} 开始")

    response = await call_next(request)

    # 在响应头中返回
    response.headers["X-Request-ID"] = request_id
    logger.info(f"[{request_id}] 完成, 状态码: {response.status_code}")

    return response
```

在路由中读取：

```python
@app.get("/debug")
async def debug(request: Request):
    """通过 request.state 获取中间件注入的数据"""
    return {"request_id": request.state.request_id}
```

### 3.2 全局异常捕获

```python
import traceback
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse


@app.middleware("http")
async def global_exception_handler_middleware(request: Request, call_next):
    """捕获所有未处理的异常，返回统一错误格式"""
    try:
        response = await call_next(request)
        return response
    except Exception as exc:
        # 记录完整堆栈
        logger.error(f"未处理的异常: {exc}\n{traceback.format_exc()}")

        # 返回统一格式（不会暴露服务器内部信息）
        return JSONResponse(
            status_code=500,
            content={
                "error": "Internal Server Error",
                "detail": "服务器内部错误，请稍后重试",
                "code": "INTERNAL_ERROR",
                # 开发环境可以返回更多信息
                # "debug": str(exc),
            },
        )
```

> ⚠️ 全局异常中间件会**覆盖** FastAPI 的默认异常处理器。如果你已经有了 `@app.exception_handler(Exception)`，注意不要重复定义。

### 3.3 请求/响应日志

```python
import time
import json
from fastapi import FastAPI, Request, Response


@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    """结构化日志：记录每个请求的详细信息"""

    start = time.perf_counter()

    # 记录请求
    log_data = {
        "method": request.method,
        "path": request.url.path,
        "query": str(request.query_params),
        "client": request.client.host if request.client else "unknown",
        "user_agent": request.headers.get("User-Agent", ""),
    }

    response: Response = await call_next(request)

    # 补充响应信息
    duration = time.perf_counter() - start
    log_data.update({
        "status": response.status_code,
        "duration_ms": round(duration * 1000, 2),
    })

    # 结构化输出（生产环境应输出到日志收集系统）
    logger.info(json.dumps(log_data, ensure_ascii=False))

    return response
```

### 3.4 请求限流（简易版）

```python
from collections import defaultdict
import time
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse


# 简单的内存限流器（生产环境请用 Redis）
class SimpleRateLimiter:
    def __init__(self, max_requests: int = 60, window: int = 60):
        self.max_requests = max_requests  # 窗口内最大请求数
        self.window = window              # 时间窗口（秒）
        self.requests: dict[str, list[float]] = defaultdict(list)

    def is_allowed(self, client_ip: str) -> bool:
        now = time.time()
        # 清除过期记录
        self.requests[client_ip] = [
            t for t in self.requests[client_ip]
            if now - t < self.window
        ]
        if len(self.requests[client_ip]) >= self.max_requests:
            return False
        self.requests[client_ip].append(now)
        return True


limiter = SimpleRateLimiter(max_requests=30, window=60)


@app.middleware("http")
async def rate_limit_middleware(request: Request, call_next):
    client_ip = request.client.host if request.client else "unknown"

    if not limiter.is_allowed(client_ip):
        return JSONResponse(
            status_code=429,
            content={
                "detail": "请求过于频繁，请稍后再试",
                "retry_after": 60,
            },
        )

    return await call_next(request)
```

## 4. CORS 中间件

### 4.1 什么是 CORS？—— 浏览器的一道安全门

CORS（Cross-Origin Resource Sharing，跨域资源共享）是浏览器内置的安全机制。它的规则很简单：**不同"源"的网页不能随意访问另一个源的 API**。

什么是"不同源"？只要**协议、域名、端口**中任意一个不一样，就是不同源：

| 前端地址 | 后端 API 地址 | 是否跨域？ | 原因 |
|----------|-------------|-----------|------|
| `http://localhost:3000` | `http://localhost:8000` | ✅ 跨域 | 端口不同 |
| `http://myapp.com` | `http://api.myapp.com` | ✅ 跨域 | 域名不同（子域名也算） |
| `https://myapp.com` | `http://myapp.com` | ✅ 跨域 | 协议不同 |
| `http://myapp.com` | `http://myapp.com/api` | ❌ 不跨域 | 同源 |

浏览器处理跨域请求的流程：

```
前端 JS: fetch("http://localhost:8000/api/users")
    │
    ▼
浏览器检查：当前页面 origin(http://localhost:3000) ≠ 目标(http://localhost:8000)
    │
    ▼
浏览器先发 OPTIONS 预检请求 → "你们允许 localhost:3000 来访问吗？"
    │
    ▼
如果服务器没返回"允许" → 浏览器直接阻止，JS 收到一个错误
如果服务器返回了正确的 CORS 头 → 浏览器放行，正常请求继续
```

> **重点**：CORS 是**浏览器**的安全策略，不是服务端的。你用 curl 或 Python 发请求**不受 CORS 限制**——这也是为什么后端开发时用 curl 测试没问题，前端却报错的原因。

### 4.2 基本配置

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# 必须在应用创建后立即添加
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",     # React 开发服务器
        "http://localhost:5173",     # Vite 开发服务器
        "https://myapp.com",         # 生产环境前端域名
    ],
    allow_credentials=True,          # 允许发送 Cookie
    allow_methods=["GET", "POST", "PUT", "DELETE", "PATCH"],  # 或 ["*"]
    allow_headers=["*"],             # 允许所有请求头
    expose_headers=[                 # 前端 JS 可读取的响应头
        "X-Process-Time",
        "X-Request-ID",
    ],
    max_age=3600,                    # 预检请求缓存（秒）
)
```

### 4.3 参数详解

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `allow_origins` | 允许的前端域名列表 | 开发环境：`["http://localhost:3000"]`<br>生产环境：具体域名，**不要用 `["*"]`** |
| `allow_origin_regex` | 用正则匹配域名 | `r"https://.*\.myapp\.com"` |
| `allow_credentials` | 是否允许携带 Cookie | `True`（需配合 `allow_origins` 具体域名） |
| `allow_methods` | 允许的 HTTP 方法 | `["GET", "POST", "PUT", "DELETE"]` |
| `allow_headers` | 允许的请求头 | `["*"]` 或具体列表 |
| `expose_headers` | JS 可读取的响应头 | 自定义头必须在此列出 |
| `max_age` | 预检缓存时间（秒） | `3600`（1 小时） |

### 4.4 CORS 配置最佳实践

```python
# ✅ 推荐：根据环境配置
import os

ENV = os.getenv("ENV", "development")

if ENV == "development":
    origins = [
        "http://localhost:3000",
        "http://localhost:5173",
        "http://127.0.0.1:3000",
    ]
else:
    origins = [
        "https://myapp.com",
        "https://admin.myapp.com",
    ]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "PATCH"],
    allow_headers=["Authorization", "Content-Type"],  # 明确列出而非用 *
    expose_headers=["X-Request-ID"],
    max_age=3600,
)
```

```python
# ❌ 危险：生产环境允许所有来源
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 任何网站都能调用你的 API！
    allow_credentials=True,  # 这个组合浏览器会直接报错
)
# 注意：allow_origins=["*"] 和 allow_credentials=True 不能同时使用
```

### 4.5 调试 CORS 问题

常见错误：

```
Access to fetch at 'http://localhost:8000/api/users' from origin
'http://localhost:3000' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present on the requested resource.
```

排查步骤：

1. **检查 CORS 中间件是否添加**：`app.add_middleware(CORSMiddleware, ...)`
2. **检查 orgin 是否在 `allow_origins` 中**：浏览器发送的 `Origin` 头必须匹配
3. **检查中间件顺序**：CORS 中间件应该最先添加
4. **检查是否同时用了 `["*"]` 和 `allow_credentials=True`**
5. **打开浏览器 DevTools Network 标签**：看 OPTIONS 预检请求的响应头

## 5. 内置中间件

FastAPI/Starlette 内置了几个常用的中间件：

### 5.1 GZipMiddleware — 响应压缩

```python
from starlette.middleware.gzip import GZipMiddleware

app.add_middleware(GZipMiddleware, minimum_size=1000)
# minimum_size: 小于此字节数的响应不压缩（默认 500）
```

### 5.2 TrustedHostMiddleware — 主机头白名单

```python
from starlette.middleware.trustedhost import TrustedHostMiddleware

app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=[
        "myapp.com",
        "*.myapp.com",        # 支持通配符
        "localhost",
    ],
)
# 防止 HTTP Host Header 攻击
```

### 5.3 HTTPSRedirectMiddleware — 强制 HTTPS

```python
from starlette.middleware.httpsredirect import HTTPSRedirectMiddleware

app.add_middleware(HTTPSRedirectMiddleware)
# 所有 HTTP 请求自动 307 重定向到 HTTPS
```

### 5.4 SessionMiddleware — 服务端 Session

```python
from starlette.middleware.sessions import SessionMiddleware

app.add_middleware(
    SessionMiddleware,
    secret_key="your-secret-key",  # 加密 session 的密钥
)

# 在路由中使用
@app.get("/set-session")
async def set_session(request: Request):
    request.session["user_id"] = 42
    return {"message": "Session set"}

@app.get("/get-session")
async def get_session(request: Request):
    return {"user_id": request.session.get("user_id")}
```

## 6. 中间件的执行顺序

这是中间件最容易被忽视但最重要的知识点。`add_middleware` 的调用顺序直接决定了中间件的执行顺序——**先添加的在最外层，后添加的在最内层**。

```python
# 洋葱模型：先添加 = 外皮，后添加 = 内芯
app.add_middleware(A)          # 第 1 个添加 → 最外层（最先处理请求，最后处理响应）
app.add_middleware(B)          # 第 2 个添加 → 中间层
app.add_middleware(C)          # 第 3 个添加 → 最内层（最后处理请求，最先处理响应）

# 请求流向（进门）：A → B → C → 路由
# 响应流向（出门）：路由 → C → B → A
```

为什么顺序重要？举个例子：
- CORS 中间件必须在最外层——因为 OPTIONS 预检请求需要在任何自定义逻辑之前被处理
- GZip 压缩也应该靠外——这样它才能压缩最终响应
- 自定义日志中间件放在中间——这样它记录的耗时才是实际业务的耗时（不含 CORS 开销）

**推荐顺序：**

```python
# 1. CORS — 最先，确保跨域预检请求能被处理
app.add_middleware(CORSMiddleware, ...)

# 2. HTTPS 重定向
app.add_middleware(HTTPSRedirectMiddleware)

# 3. TrustedHost
app.add_middleware(TrustedHostMiddleware, ...)

# 4. GZip — 压缩（在最后，因为需要在响应返回前压缩）
app.add_middleware(GZipMiddleware, minimum_size=1000)

# 5. 自定义中间件
app.add_middleware(request_id_middleware)
app.add_middleware(logging_middleware)
```

## 7. 综合示例

```python
"""
中间件综合演示 — 请求 ID + 日志 + 计时 + CORS
"""
import uuid
import time
import logging
import json
from fastapi import FastAPI, Request
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import JSONResponse

# ── 日志配置 ──
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger("api")

app = FastAPI(title="中间件演示 API")

# ── 1. CORS — 最外层 ──
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
    expose_headers=["X-Request-ID", "X-Process-Time"],
)

# ── 2. 全局异常捕获 ──
@app.middleware("http")
async def exception_middleware(request: Request, call_next):
    try:
        return await call_next(request)
    except Exception as e:
        logger.exception(f"未处理的异常: {e}")
        return JSONResponse(
            status_code=500,
            content={"detail": "服务器内部错误"},
        )

# ── 3. 请求 ID ──
@app.middleware("http")
async def request_id_middleware(request: Request, call_next):
    request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))
    request.state.request_id = request_id
    response = await call_next(request)
    response.headers["X-Request-ID"] = request_id
    return response

# ── 4. 结构化日志 ──
@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    start = time.perf_counter()
    response = await call_next(request)
    duration = (time.perf_counter() - start) * 1000

    log_entry = {
        "request_id": getattr(request.state, "request_id", "-"),
        "method": request.method,
        "path": request.url.path,
        "status": response.status_code,
        "duration_ms": round(duration, 2),
    }
    logger.info(json.dumps(log_entry, ensure_ascii=False))
    return response

# ── 端点 ──
@app.get("/")
async def root():
    return {"message": "Hello"}

@app.get("/error")
async def error():
    """测试全局异常捕获"""
    raise ValueError("模拟一个错误")

@app.get("/slow")
async def slow():
    """测试耗时记录"""
    time.sleep(0.5)
    return {"message": "slow response"}
```
