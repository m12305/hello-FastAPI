# 2.2 请求头、Cookie 与表单

> API 不只是处理 JSON —— 请求头、Cookie、表单数据和文件上传同样是日常开发的必备技能。

---

## 1. 数据来源全景图

在前面的章节中，我们主要处理了两种数据：URL 路径中的参数（`/items/42`）和 URL 查询参数（`?page=1`）。但一个 HTTP 请求携带数据的方式远不止这些。

可以把一个 HTTP 请求想象成一个**快递包裹**：
- **快递面单（请求行）**：写着收件地址（URL 路径）和配送方式（GET/POST）
- **包裹标签（请求头）**：贴了各种标签——"易碎品"、"加急"、"签名确认"——提供关于请求的元信息
- **包裹内容（请求体）**：里面装的实际货物——可能是 JSON 文件、表单数据、上传的图片

不同的数据放在不同的位置，FastAPI 为每种位置提供了对应的提取工具：

```
HTTP 请求
├── 请求行    → GET /items/42?page=1 HTTP/1.1
│   ├── 方法  → @app.get()
│   ├── 路径  → Path()  — 路径参数 /items/{item_id}
│   └── 查询  → Query() — 查询参数 ?page=1
│
├── 请求头    → Header() — Content-Type, Authorization, User-Agent...
├── Cookie    → Cookie() — session_id, csrf_token...
│
└── 请求体
    ├── JSON       → Pydantic Model
    ├── 表单       → Form() — application/x-www-form-urlencoded
    ├── 文件       → File() / UploadFile — multipart/form-data
    └── 原始数据   → Request.body()
```

> 本章重点讲 **Header()**、**Cookie()**、**Form()** 和 **File()**——这四种工具覆盖了除 JSON 外的所有常见数据来源。

## 2. `Header()` — 读取请求头

请求头是附加在 HTTP 请求上的**键值对元数据**。它们不直接参与业务逻辑，但提供了关键的上下文信息——比如你是谁（Authorization）、你用什么客户端（User-Agent）、你希望返回什么格式的数据（Accept）。

### 为什么需要读取请求头？

最常见的场景：
- **认证**：从 `Authorization` 头中提取 JWT Token 或 API Key
- **追踪**：从 `X-Request-ID` 头中获取请求 ID，串联日志
- **路由**：根据 `Accept-Language` 返回不同语言的内容
- **安全**：根据 `User-Agent` 或 `X-Forwarded-For` 识别恶意请求

```python
from fastapi import FastAPI, Header

app = FastAPI()


@app.get("/whoami")
async def whoami(
    # 读取单个请求头
    user_agent: str | None = Header(default=None),
    # HTTP 头中的连字符自动转为下划线
    # User-Agent → user_agent

    # 必选的请求头
    x_api_key: str = Header(default=..., alias="X-API-Key"),
    # 注意：FastAPI 会自动做大小写不敏感匹配

    # 读取列表值（某些头可能出现多次）
    accept_language: str | None = Header(default=None),
):
    return {
        "user_agent": user_agent,
        "api_key_provided": bool(x_api_key),
        "accept_language": accept_language,
    }
```

### Header 参数详解

```python
@app.get("/headers")
async def read_headers(
    # 有默认值 → 可选
    content_type: str | None = Header(default=None),

    # ... (Ellipsis) → 必选
    authorization: str = Header(default=...),

    # convert_underscores: 是否自动转换下划线为连字符（默认 True）
    # 如果设为 False，x_token 匹配的是 X-Token 而非 X_Token
    x_token: str | None = Header(default=None, convert_underscores=True),
):
    return {"content_type": content_type}
```

### 常用的请求头

| Header | 说明 | 示例值 |
|--------|------|--------|
| `User-Agent` | 客户端标识 | `Mozilla/5.0 ...` |
| `Content-Type` | 请求体格式 | `application/json` |
| `Authorization` | 认证凭证 | `Bearer eyJhbG...` |
| `Accept` | 接受的响应格式 | `application/json` |
| `Accept-Language` | 语言偏好 | `zh-CN, en;q=0.9` |
| `X-Forwarded-For` | 客户端真实 IP（代理后） | `192.168.1.1` |
| `X-Request-ID` | 请求追踪 ID | `550e8400-e29b-...` |

## 3. `Cookie()` — 读取 Cookie

Cookie 本质上也是一种请求头——它存储在 `Cookie` 头中，以 `key1=value1; key2=value2` 的格式发送。但 Cookie 有独特的**安全属性**和**自动携带**机制，所以 FastAPI 为它提供了专门的 `Cookie()` 函数。

### Cookie 的工作原理（简述）

当服务器通过 `Set-Cookie` 响应头告诉浏览器"记住这个值"，浏览器就会把它存在本地。之后向同一域名发请求时，浏览器会**自动**把 Cookie 附带在 `Cookie` 请求头中——不需要 JavaScript 手动操作。这是 Cookie 和普通 Header 最大的区别：Cookie 是浏览器自动管理的。

```
1. 服务器响应:  Set-Cookie: session_id=abc123; HttpOnly; Secure
2. 浏览器存储:  域名=myapp.com, key=session_id, value=abc123
3. 下次请求:    浏览器自动附加 Cookie: session_id=abc123
4. 服务器读取:  Cookie(default=None) → "abc123"
```

```python
from fastapi import FastAPI, Cookie

app = FastAPI()


@app.get("/read-cookies")
async def read_cookies(
    session_id: str | None = Cookie(default=None),
    csrf_token: str | None = Cookie(default=None),
    # Cookie 名中的连字符自动转为下划线（和 Header 一样）：
    # 浏览器中的 csrf-token → Python 中的 csrf_token
):
    return {
        "session_id": session_id,
        "csrf_token": csrf_token,
    }
```

### Cookie vs Header 对比

| | Cookie | Header |
|------|--------|--------|
| 数据来源 | `Cookie` 请求头中的键值对 | 任意 HTTP 头 |
| 自动携带 | 浏览器自动附带同域 Cookie | 需手动设置 |
| 安全性 | 可设 HttpOnly、Secure、SameSite | 完全由客户端控制 |
| 典型用途 | 会话标识、用户偏好 | API Key、JWT Token |
| 在 FastAPI 中 | `Cookie()` 函数 | `Header()` 函数 |

> **最佳实践**：API 认证优先使用 `Authorization` 头（JWT 等），而非 Cookie。Cookie 更适合前后端不分离的传统 Web 应用。

### 设置 Cookie（在响应中）

```python
from fastapi import FastAPI, Response


@app.post("/login")
async def login(response: Response):
    # 在响应中设置 Cookie
    response.set_cookie(
        key="session_id",
        value="abc123",
        max_age=3600,          # 有效期（秒）
        httponly=True,         # JS 不可访问
        secure=True,           # 仅 HTTPS 发送
        samesite="lax",        # CSRF 防护
    )
    return {"message": "登录成功"}


@app.post("/logout")
async def logout(response: Response):
    # 删除 Cookie
    response.delete_cookie(key="session_id")
    return {"message": "已登出"}
```

## 4. `Form()` — 处理表单提交

表单是 Web 最古老的交互方式——在 JSON API 流行之前，几乎所有数据提交都是用 HTML `<form>` 标签完成的。如今表单数据主要出现在两个场景：
- **OAuth2 密码模式**：FastAPI 内置的登录表单使用 `application/x-www-form-urlencoded` 格式
- **传统 HTML 页面**：后台管理界面中的创建/编辑表单

表单数据的格式和 JSON 不同：JSON 是 `{"key": "value"}`，表单是 `key=value&key2=value2`（和 URL 查询参数一样，只是放在请求体中）。

```python
from fastapi import FastAPI, Form

app = FastAPI()


@app.post("/login")
async def login(
    # Form() 用起来和 Query() 几乎一样，只是数据从请求体读取而非 URL
    username: str = Form(default=...),
    password: str = Form(default=...),
    grant_type: str | None = Form(default=None),
):
    """处理 application/x-www-form-urlencoded 格式的登录请求"""
    # ⚠️ 生产环境中密码需要哈希验证！
    return {"username": username, "grant_type": grant_type}


@app.post("/register")
async def register(
    username: str = Form(min_length=3, max_length=30),
    password: str = Form(min_length=8),
    email: str = Form(pattern=r"^[^@]+@[^@]+\.[^@]+$"),
):
    """Form 也可以使用校验参数——min_length、pattern 等和 Query/Path 完全一样"""
    return {
        "username": username,
        "email": email,
    }
```

### 表单 vs JSON 请求体 —— 关键区别

FastAPI 的一个端点只能处理**一种**请求体格式。这是因为 HTTP 的 `Content-Type` 头只能有一个值：要么是 `application/json`（JSON），要么是 `application/x-www-form-urlencoded`（表单），要么是 `multipart/form-data`（文件+表单混合）。

```python
# ❌ 不能混用 Form 和 Pydantic Model 在同一个端点
# FastAPI 不知道应该按表单解析还是按 JSON 解析

# @app.post("/broken")
# async def broken(
#     name: str = Form(...),    # ← 要求表单格式
#     item: ItemCreate,          # ← 要求 JSON 格式 → 冲突！
# ):
#     ...

# ✅ 正确做法：全部用 Form
@app.post("/form-only")
async def form_only(
    name: str = Form(...),
    description: str = Form(...),
    price: float = Form(gt=0),
):
    return {"name": name, "price": price}


# ✅ 或全部用 Pydantic Model（推荐——大多数场景用这个）
@app.post("/json-only")
async def json_only(item: ItemCreate):
    return {"item": item}
```

> ⚠️ **重要**：一个端点要么用 Form，要么用 JSON 请求体，不能混用。**大多数情况下用 JSON + Pydantic Model**，只有在对接 OAuth2 或 HTML 表单时才用 `Form()`。

## 5. `File()` 与 `UploadFile` — 文件上传

文件上传可能是 Web 开发中最容易出错的功能之一——文件太大怎么办？类型不对怎么办？文件名有恶意字符怎么办？FastAPI 提供了两种方式接收文件，各有适用场景。

### 两种文件接收方式的本质区别

- **`bytes`**：FastAPI 把整个文件内容读进内存，作为一个字节数组传给你的函数。简单直接，但文件太大会撑爆内存。
- **`UploadFile`**：FastAPI 把文件内容写入磁盘临时文件，给你一个文件句柄。你可以分块读取，类似 Python 的 `open()`。适合所有场景，是官方推荐的方式。

### 5.1 小文件上传 — `bytes`

```python
from fastapi import FastAPI, File

app = FastAPI()


@app.post("/upload/small")
async def upload_small_file(
    file: bytes = File(default=...),
):
    """整个文件内容读入内存——只适合 < 1MB 的小文件"""
    return {
        "filename": "unknown",  # bytes 模式下拿不到原始文件名
        "size": len(file),
        "content_preview": file[:100].decode("utf-8", errors="replace"),
    }
```

### 5.2 标准文件上传 — `UploadFile`（推荐）

```python
from fastapi import FastAPI, File, UploadFile

app = FastAPI()


@app.post("/upload")
async def upload_file(
    file: UploadFile = File(default=..., description="要上传的文件"),
):
    """UploadFile 是官方推荐的方式，提供完整的文件信息"""
    content = await file.read()  # 异步读取文件内容

    return {
        "filename": file.filename,         # 客户端上传时的原始文件名
        "content_type": file.content_type, # 文件的 MIME 类型（如 image/png）
        "size": len(content),              # 文件大小（字节）
        "headers": dict(file.headers),     # multipart 部分的额外头信息
    }
```

### `bytes` vs `UploadFile` 对比

| | `bytes` | `UploadFile` |
|------|---------|-------------|
| 内存占用 | 整个文件在内存中 | 可配置磁盘缓冲（大文件自动写入临时文件） |
| 文件名 | ❌ 无法获取 | ✅ `file.filename` |
| MIME 类型 | ❌ 无法获取 | ✅ `file.content_type` |
| 大文件 | ⚠️ 内存溢出风险 | ✅ 分块读取，不会撑爆内存 |
| 异步支持 | 同步 | ✅ `await file.read()` |
| 推荐场景 | 小文件（< 1MB） | **通用场景（推荐）** |

### 5.3 完整的文件上传处理（含校验）

真实项目中的文件上传远不止接收和保存——还需要校验类型、限制大小、生成安全文件名。下面是一个生产级的图片上传接口：

```python
import os
import uuid
from pathlib import Path

from fastapi import FastAPI, UploadFile, File, HTTPException

app = FastAPI()

UPLOAD_DIR = Path("./uploads")
UPLOAD_DIR.mkdir(exist_ok=True)  # 确保上传目录存在


@app.post("/upload/image")
async def upload_image(file: UploadFile = File(...)):
    # 1. 校验文件类型——根据 MIME 类型拒绝非图片文件
    ALLOWED_TYPES = {
        "image/jpeg": [".jpg", ".jpeg"],
        "image/png": [".png"],
        "image/gif": [".gif"],
        "image/webp": [".webp"],
    }

    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(
            status_code=400,
            detail=f"不支持的文件类型：{file.content_type}。"
                   f"仅支持：{', '.join(ALLOWED_TYPES.keys())}",
        )

    # 2. 校验文件大小——防止用户上传超大文件
    content = await file.read()
    MAX_SIZE = 5 * 1024 * 1024  # 5MB

    if len(content) > MAX_SIZE:
        raise HTTPException(
            status_code=400,
            detail=f"文件大小 {len(content) / 1024 / 1024:.1f}MB "
                   f"超过上限 {MAX_SIZE / 1024 / 1024:.0f}MB",
        )

    # 3. 生成安全的文件名——用 UUID 而非用户提供的文件名
    #    原因：用户文件名可能包含 ../ 等路径遍历字符，存在安全风险
    extension = ALLOWED_TYPES[file.content_type][0]  # 根据类型选后缀
    safe_filename = f"{uuid.uuid4().hex}{extension}"

    # 4. 保存文件
    file_path = UPLOAD_DIR / safe_filename
    file_path.write_bytes(content)

    return {
        "message": "上传成功",
        "original_filename": file.filename,
        "saved_filename": safe_filename,
        "size": len(content),
        "url": f"/static/{safe_filename}",  # 后续可配合 StaticFiles 挂载
    }
```

### 5.4 多文件上传

```python
from typing import List


@app.post("/upload/multiple")
async def upload_multiple_files(
    # 方式 1：用 List[UploadFile] 接收多个文件
    files: List[UploadFile] = File(default=..., description="多个文件"),
):
    results = []
    for file in files:
        content = await file.read()
        results.append({
            "filename": file.filename,
            "size": len(content),
        })

    return {"uploaded": len(results), "files": results}


@app.post("/upload/multiple-single")
async def upload_multiple_single_param(
    # 方式 2：同一个参数名传多个文件
    files: List[UploadFile] = File(...),
):
    """对应 HTML 表单 <input type='file' name='files' multiple>"""
    return {"count": len(files)}
```

### 5.5 表单 + 文件混合上传

一个常见的场景：用户填写商品信息（名称、价格等）的同时上传商品图片。此时请求体的 `Content-Type` 是 `multipart/form-data`——表单字段和文件字段混合在一起。

```python
from fastapi import FastAPI, File, UploadFile, Form


@app.post("/products/with-image")
async def create_product_with_image(
    # 表单字段——用 Form()
    name: str = Form(..., min_length=1, max_length=100),
    price: float = Form(..., gt=0),
    category: str = Form(...),

    # 文件字段——用 File()
    image: UploadFile = File(..., description="商品主图"),
    gallery: List[UploadFile] = File(default=[], description="商品图集（可选）"),
):
    image_content = await image.read()

    gallery_info = []
    for img in gallery:
        gallery_info.append({
            "filename": img.filename,
            "size": len(await img.read()),
        })

    return {
        "product": {"name": name, "price": price, "category": category},
        "image": {"filename": image.filename, "size": len(image_content)},
        "gallery": gallery_info,
    }
```

> ⚠️ 混合上传的关键点：表单字段用 `Form()`，文件字段用 `File()`。请求的 Content-Type 必须是 `multipart/form-data`（浏览器在提交含文件的表单时会自动设置）。

## 6. 完整示例：文件管理系统

```python
"""
文件管理 API — 综合运用 Header、Cookie、Form、File
"""
import os
import uuid
import hashlib
from datetime import datetime
from pathlib import Path
from typing import List

from fastapi import (
    FastAPI, File, UploadFile, Form, Header,
    Cookie, HTTPException, Response,
)

app = FastAPI(title="文件管理系统")

UPLOAD_DIR = Path("./uploads")
UPLOAD_DIR.mkdir(exist_ok=True)

# 模拟的文件数据库
files_db: dict[str, dict] = {}


def calculate_md5(content: bytes) -> str:
    return hashlib.md5(content).hexdigest()


@app.post("/files/upload")
async def upload_file(
    file: UploadFile = File(..., description="要上传的文件"),
    description: str = Form(default="", description="文件描述"),
    tags: str = Form(default="", description="标签（逗号分隔）"),

    # 请求头：用户标识
    x_user_id: str = Header(default="anonymous", alias="X-User-ID"),

    # Cookie：追踪会话
    session_id: str | None = Cookie(default=None),
):
    """
    上传文件，保存元数据。

    Headers:
    - X-User-ID: 上传者 ID（默认 anonymous）

    Cookies:
    - session_id: 用户会话 ID
    """
    # 校验文件大小
    content = await file.read()
    MAX_SIZE = 10 * 1024 * 1024  # 10MB

    if len(content) > MAX_SIZE:
        raise HTTPException(status_code=400, detail="文件不能超过 10MB")

    # 保存文件
    file_id = uuid.uuid4().hex
    extension = Path(file.filename).suffix if file.filename else ""
    safe_name = f"{file_id}{extension}"
    file_path = UPLOAD_DIR / safe_name
    file_path.write_bytes(content)

    # 记录元数据
    metadata = {
        "id": file_id,
        "original_name": file.filename,
        "saved_name": safe_name,
        "size": len(content),
        "content_type": file.content_type,
        "md5": calculate_md5(content),
        "description": description,
        "tags": [t.strip() for t in tags.split(",") if t.strip()],
        "uploader": x_user_id,
        "session_id": session_id,
        "uploaded_at": datetime.now().isoformat(),
    }
    files_db[file_id] = metadata

    return {
        "message": "上传成功",
        "file_id": file_id,
        "metadata": metadata,
    }


@app.get("/files/")
async def list_files(
    x_user_id: str = Header(default=..., alias="X-User-ID"),
):
    """列出当前用户上传的所有文件"""
    user_files = [
        f for f in files_db.values()
        if f["uploader"] == x_user_id
    ]
    return {"count": len(user_files), "files": user_files}


@app.get("/files/{file_id}")
async def get_file_info(file_id: str):
    """获取文件元数据"""
    if file_id not in files_db:
        raise HTTPException(status_code=404, detail="文件不存在")
    return files_db[file_id]
```

## 7. 常见问题排查

### 问题 1：上传大文件时内存溢出

```python
# ❌ 全部读入内存
content = await file.read()

# ✅ 使用临时文件缓冲
# UploadFile 默认会将大文件写入磁盘临时文件
# 可以通过 file.read 访问底层的 Python 文件对象
async with open("output.bin", "wb") as out:
    while chunk := await file.read(1024 * 1024):  # 每次读 1MB
        out.write(chunk)
```

### 问题 2：表单和 JSON 混用报错

```python
# ❌ 错误：同时使用 Form 和 Pydantic Model
# FastAPI 不知道如何解析请求体

# ✅ 方案 1：全用 Form
@app.post("/create")
async def create(
    name: str = Form(...),
    age: int = Form(...),
):
    ...

# ✅ 方案 2：全用 Pydantic（推荐）
@app.post("/create")
async def create(data: UserCreate):
    ...
```

### 问题 3：文件名安全问题

```python
# ❌ 直接使用用户提供的文件名
file_path = UPLOAD_DIR / file.filename  # 可能包含 ../ 路径遍历攻击！

# ✅ 生成安全的文件名
import uuid
safe_name = f"{uuid.uuid4().hex}{Path(file.filename).suffix}"
file_path = UPLOAD_DIR / safe_name
```

"""练习：头像上传服务

   - file: UploadFile（图片文件）
   - crop_x, crop_y, crop_w, crop_h: int Form 参数（裁剪坐标，可选）
   - X-User-ID: 请求头，必选

2. 校验逻辑：

   - 文件大小不超过 2MB
   - 如果 content_type 与实际文件头不匹配，拒绝

3. 文件处理：
   - 生成 UUID 文件名
   - 保存到 ./avatars/{user_id}/ 目录
   - 如果目录不存在，自动创建

4. 返回：
   - 上传成功的消息
   - 文件 URL
