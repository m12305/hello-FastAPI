# FastAPI 学习方法

> 本方法适用于学习路线中的每一个章节。请严格按照此方法进行学习，确保每一步都扎实掌握。

---

## 六步学习法

```
预览 → 学习 → 练习 → 反思 → 构建 → 复习
```

---

## 第一步：预览（5-10 分钟）

在深入学习之前，先快速浏览本章节的所有内容标题和代码示例，建立全局认知。

**操作清单：**
- [ ] 通读本章节的目录和所有小标题
- [ ] 快速浏览所有代码示例，不要求理解细节
- [ ] 用一句话写下：本章节要解决什么问题？
- [ ] 思考：这个知识点在我之前的学习/工作中出现过吗？

**产出：** 在「学习进度」的对应章节中写下你的 **一句话总结** 和 **初始疑问**。

---

## 第二步：学习（30-60 分钟）

这是核心输入阶段。跟着教程动手敲每一行代码，**不要复制粘贴**。

**操作清单：**
- [ ] 打开编辑器，**手动敲入**教程中的所有代码
- [ ] 运行代码，观察输出结果是否与教程一致
- [ ] 如果报错，先尝试自己排查 3 分钟，再查资料
- [ ] 对每个新概念（函数、装饰器、类），用自己的话做注释
- [ ] 记录下你遇到的问题和解决方案

**产出：** 一个可以运行的代码文件 + 代码注释。

---

## 第三步：练习（20-40 分钟）

变着花样改代码，验证你真的理解了。

**操作清单：**
- [ ] **改动参数**：把教程中的参数改掉，看会发生什么
- [ ] **故意写错**：故意制造错误，观察 FastAPI 的报错信息
- [ ] **扩展功能**：在原例子上加一个额外的功能点
- [ ] **对比写法**：如果同一功能有多种写法，都试一遍

**产出：** 至少 3 个你"玩坏过"的代码变体。

---

## 第四步：反思（10-15 分钟）

用自己的话总结，而不是复述教程。

**操作清单：**
- [ ] 用费曼学习法：假装要给一个新手讲这个知识点，写一段 200 字以内的解释
- [ ] 画出知识点的关系图（可以用 mermaid 或手绘）
- [ ] 回答三个问题：
  - **What**：这个知识点是什么？
  - **Why**：为什么需要它？（不用它会怎样？）
  - **How**：怎么用它？（给一个最小可用示例）
- [ ] 对比：它和我之前学过的其他框架/工具相似吗？区别是什么？

**产出：** 在「学习进度」中填写 **反思笔记**。

---

## 第五步：构建（30-60 分钟）

脱离教程，自己造一个小轮子。

**操作清单：**
- [ ] 设计一个与本知识点相关的小功能（不要求完整项目）
- [ ] 不使用教程中的案例，自己想一个真实场景
- [ ] 先写伪代码或注释描述你要做什么，再逐行实现
- [ ] 可以结合之前学过的知识点

**灵感来源：**
- 把你平时用的某个网站/工具的功能拆解成 API
- 做一个简化版的 Todo、博客、记账、打卡等
- 把某个命令行工具封装成 API 接口

**产出：** 一个独立的 `.py` 文件，包含你独立实现的功能。

---

## 第六步：复习（10 分钟 + 定期）

遗忘曲线告诉我们：不复习 = 白学。

**操作清单：**
- [ ] 当天晚上：花 5 分钟回顾今天写了什么代码
- [ ] 第二天早上：尝试在不看代码的情况下，写出核心 API 的骨架
- [ ] 每周日：回顾本周所有章节的反思笔记
- [ ] 每完成一个阶段：重新做一遍该阶段的"阶段实战项目"

**产出：** 在学习进度中标注复习日期。

---

## 遇到困难怎么办？

1. **先自己调试 3-5 分钟** — 仔细读报错信息，用 `print()` / `logging` 排查
2. **查官方文档** — FastAPI 官方文档非常优秀，优先查 [fastapi.tiangolo.com](https://fastapi.tiangolo.com)
3. **看源码** — 对某个函数不理解？直接 `Ctrl+Click` 跳进去看源码
4. **记录问题** — 在「学习进度」的"踩坑记录"中记下来，避免重复踩坑
5. **求助社区** — 带着你已经尝试过的方案去 Stack Overflow / V2EX / 知乎提问

## 学习节奏建议

| 节奏 | 每日时间 | 适合人群 |
|------|---------|---------|
| **冲刺型** | 3-4 小时/天 | 有经验的开发者，快速上手 |
| **稳健型** | 1-2 小时/天 | 有一定基础，稳步推进 |
| **长期型** | 0.5-1 小时/天 | 业余时间学习，细水长流 |

> 建议选择**稳健型**节奏，每个章节用 1-2 天完成。每周至少完成 3 个章节。

## 学习环境准备

确保你的开发环境已就绪：

```bash
# Python >= 3.8
python --version

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows

# 安装核心依赖
pip install fastapi uvicorn[standard]
```
