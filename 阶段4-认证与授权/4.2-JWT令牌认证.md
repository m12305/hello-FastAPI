## 1. JWT 是什么？

JWT（JSON Web Token）是一个开放标准（RFC 7519），定义了一种紧凑、自包含的方式在各方之间安全传输信息。

### JWT 的结构

```
eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxMjM0NTY3ODkwIn0.dozjgNryP4J3jVmNHl0w5N_XgL0N3A

┌──────────────────────┬──────────────────────┬──────────────────────┐
│       Header          │       Payload         │      Signature        │
│      头               │       载荷             │      签名             │
├──────────────────────┼──────────────────────┼──────────────────────┤
│ {                     │ {                     │ HMACSHA256(          │
│   "alg": "HS256",     │   "sub": "123456",    │   base64(header)     │
│   "typ": "JWT"        │   "name": "张三",      │   + "." +            │
│ }                     │   "exp": 1718000000,   │   base64(payload),   │
│                       │   "role": "user"      │   secret_key         │
│                       │ }                     │ )                    │
└──────────────────────┴──────────────────────┴──────────────────────┘
```

| 部分 | 内容 | 说明 |
|------|------|------|
| **Header** | 算法 + 类型 | 告诉别人这是什么 token、用什么签名 |
| **Payload** | 数据（Claims） | 用户 ID、角色、过期时间等（Base64 编码，非加密！） |
| **Signature** | 签名 | 用密钥对前两部分签名，防止篡改 |

> ⚠️ **关键理解：** JWT 的 Payload 只是 Base64 编码，不是加密！任何人都可以解码看到内容。所以**绝对不要在 JWT 中放密码等敏感信息**。

## 2. JWT 的核心 Claims

```python
{
    # ── 注册 Claims（标准字段）──
    "sub": "user_123",           # Subject：主体（通常是用户 ID）
    "iss": "my-app",             # Issuer：签发者
    "aud": "my-api",             # Audience：接收方
    "exp": 1718000000,           # Expiration：过期时间（UNIX 时间戳）
    "iat": 1717900000,           # Issued At：签发时间
    "nbf": 1717900000,           # Not Before：在此之前无效

    # ── 自定义 Claims ──
    "username": "zhangsan",
    "role": "admin",
}
```

**最常用的 Claim：**
- `sub`：用户唯一标识
- `exp`：过期时间（**生产环境必须设置！**）
- `role` / `scope`：权限信息

### 4.2 `auth.py`——认证核心模块

```python
"""auth.py — JWT 生成/验证、密码哈希、get_current_user 依赖"""
from datetime import datetime, timedelta
from jose import JWTError, jwt
from passlib.context import CryptContext
from fastapi import Depends, HTTPException, status
from fastapi.security import OAuth2PasswordBearer
from sqlalchemy.orm import Session
from database import get_db
from models import User

# ═══════════ 配置 ═══════════
SECRET_KEY = "your-secret-key-change-in-production-use-env-var"  # ← 生产环境用环境变量
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30   # Access Token 有效期
REFRESH_TOKEN_EXPIRE_DAYS = 7      # Refresh Token 有效期

# ═══════════ 密码哈希 ═══════════
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")


def verify_password(plain_password: str, hashed_password: str) -> bool:
    """验证密码"""
    return pwd_context.verify(plain_password, hashed_password)


def get_password_hash(password: str) -> str:
    """生成密码哈希"""
    return pwd_context.hash(password)


# ═══════════ JWT 生成 ═══════════

def create_access_token(data: dict, expires_delta: timedelta | None = None) -> str:
    """生成 Access Token"""
    to_encode = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(minutes=15))
    to_encode.update({"exp": expire})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


def create_refresh_token(data: dict) -> str:
    """生成 Refresh Token（有效期更长）"""
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(days=REFRESH_TOKEN_EXPIRE_DAYS)
    to_encode.update({"exp": expire, "type": "refresh"})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


# ═══════════ JWT 验证 ═══════════

def decode_access_token(token: str) -> dict:
    """解码并验证 Access Token"""
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except JWTError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="无效的 Token",
        )


# ═══════════ OAuth2 方案 ═══════════
# 告诉 FastAPI：Token 从 Authorization 头的 Bearer 字段获取
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/login")


# ═══════════ 获取当前用户（最核心的依赖） ═══════════

def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db),
) -> User:
    """从 Token 中解析当前登录用户——所有受保护端点共用此依赖"""
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="无法验证凭证",
        headers={"WWW-Authenticate": "Bearer"},
    )

    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id: int = payload.get("sub")
        if user_id is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception

    user = db.get(User, int(user_id))
    if user is None:
        raise credentials_exception
    if not user.is_active:
        raise HTTPException(status_code=400, detail="用户已被停用")

    return user


def get_current_active_superuser(
    current_user: User = Depends(get_current_user),
) -> User:
    """获取当前超级管理员——嵌套依赖"""
    if current_user.role != "admin":
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="需要管理员权限",
        )
    return current_user
```

### 4.3 `schemas.py`——Pydantic 模型

```python
"""schemas.py — 认证相关的 Pydantic 模型"""
from pydantic import BaseModel, EmailStr, Field


class UserCreate(BaseModel):
    """注册请求体"""
    username: str = Field(min_length=3, max_length=50)
    email: EmailStr
    password: str = Field(min_length=8, max_length=100)


class UserResponse(BaseModel):
    """用户信息响应（无密码）"""
    model_config = {"from_attributes": True}
    id: int
    username: str
    email: str
    is_active: bool
    role: str


class Token(BaseModel):
    """Token 响应"""
    access_token: str
    refresh_token: str
    token_type: str = "bearer"


class TokenRefresh(BaseModel):
    """刷新 Token 的请求体"""
    refresh_token: str


class LoginRequest(BaseModel):
    """登录请求体（可选——也可以用 OAuth2PasswordRequestForm）"""
    username: str
    password: str
```

### 4.4 路由实现

```python
"""main.py — 认证路由"""
from datetime import timedelta
from fastapi import FastAPI, Depends, HTTPException, status
from fastapi.security import OAuth2PasswordRequestForm
from sqlalchemy.orm import Session
from database import engine, Base, get_db
from models import User
from schemas import UserCreate, UserResponse, Token, TokenRefresh
from auth import (
    get_password_hash, verify_password,
    create_access_token, create_refresh_token,
    decode_access_token, get_current_user,
    ACCESS_TOKEN_EXPIRE_MINUTES,
)

app = FastAPI(title="JWT Auth Demo")


@app.on_event("startup")
def startup():
    Base.metadata.create_all(bind=engine)


# ═══════════════════════════════════════
# 注册
# ═══════════════════════════════════════

@app.post("/register", response_model=UserResponse, status_code=201, tags=["认证"])
def register(user_in: UserCreate, db: Session = Depends(get_db)):
    """用户注册"""
    # 检查用户名是否已存在
    existing = db.query(User).filter(
        (User.username == user_in.username) | (User.email == user_in.email)
    ).first()
    if existing:
        raise HTTPException(
            status_code=400,
            detail="用户名或邮箱已被注册",
        )

    user = User(
        username=user_in.username,
        email=user_in.email,
        hashed_password=get_password_hash(user_in.password),
        role="user",  # 默认角色
    )
    db.add(user)
    db.commit()
    db.refresh(user)
    return user


# ═══════════════════════════════════════
# 登录（获取 Token）
# ═══════════════════════════════════════

@app.post("/login", response_model=Token, tags=["认证"])
def login(
    form_data: OAuth2PasswordRequestForm = Depends(),  # ← FastAPI 内置表单解析
    db: Session = Depends(get_db),
):
    """
    登录——验证用户名密码，返回 JWT Token

    这个端点就是 oauth2_scheme 中 tokenUrl="/login" 指向的端点。
    OAuth2PasswordRequestForm 期望的请求格式:
      grant_type=password&username=xxx&password=xxx
    （用表单格式，不是 JSON！Swagger 的 /docs 会自动处理。）
    """
    # 查找用户
    user = db.query(User).filter(User.username == form_data.username).first()
    if not user or not verify_password(form_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户名或密码错误",
            headers={"WWW-Authenticate": "Bearer"},
        )

    # 生成 Token
    access_token = create_access_token(
        data={"sub": str(user.id), "role": user.role},
        expires_delta=timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES),
    )
    refresh_token = create_refresh_token(data={"sub": str(user.id)})

    return {
        "access_token": access_token,
        "refresh_token": refresh_token,
        "token_type": "bearer",
    }


# ═══════════════════════════════════════
# 刷新 Token
# ═══════════════════════════════════════

@app.post("/refresh", response_model=Token, tags=["认证"])
def refresh_token(token_in: TokenRefresh, db: Session = Depends(get_db)):
    """用 Refresh Token 换取新的 Access Token"""
    try:
        payload = decode_access_token(token_in.refresh_token)
        if payload.get("type") != "refresh":
            raise HTTPException(status_code=401, detail="不是 Refresh Token")
    except Exception:
        raise HTTPException(status_code=401, detail="无效的 Refresh Token")

    user_id = payload.get("sub")
    user = db.get(User, int(user_id))
    if not user:
        raise HTTPException(status_code=401, detail="用户不存在")

    # 生成新的 Token 对
    new_access = create_access_token(
        data={"sub": str(user.id), "role": user.role},
        expires_delta=timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES),
    )
    new_refresh = create_refresh_token(data={"sub": str(user.id)})

    return {
        "access_token": new_access,
        "refresh_token": new_refresh,
        "token_type": "bearer",
    }


# ═══════════════════════════════════════
# 受保护的路由
# ═══════════════════════════════════════

@app.get("/me", response_model=UserResponse, tags=["用户"])
def read_current_user(current_user: User = Depends(get_current_user)):
    """获取当前登录用户信息（需要 Bearer Token）"""
    return current_user


@app.get("/admin", tags=["管理员"])
def admin_panel(admin: User = Depends(get_current_active_superuser)):
    """管理员面板——仅 admin 可访问"""
    return {"message": f"欢迎管理员 {admin.username}", "users_count": 1200}
```

## 5. Access Token vs Refresh Token

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  Access Token                    Refresh Token            │
│  ─────────────                   ─────────────             │
│  短期有效（15-30分钟）            长期有效（7-30天）         │
│  用于访问 API                   仅用于获取新 Access Token   │
│  每次请求都带                    不频繁使用                 │
│  被盗后影响时间短                 被盗后风险大               │
│                                                          │
└──────────────────────────────────────────────────────────┘

使用流程:
  登录 → 同时拿到 Access + Refresh
          │
          ├── 请求 API：带 Access Token
          │
          └── Access 过期 → 用 Refresh 换新的 Access + Refresh
```

## 6. Token 存储策略（前端视角）

| 存储位置 | 优点 | 缺点 |
|----------|------|------|
| **localStorage** | 方便，不受页面关闭影响 | 易受 XSS 攻击 |
| **sessionStorage** | 关闭标签即清除 | 重新打开需要重新登录 |
| **HttpOnly Cookie** | 防 XSS | 需要后端配合 + 防 CSRF |
| **内存变量** | 最安全 | 刷新页面丢失 |

> 对于 SPA 应用，推荐 Access Token 存内存 + Refresh Token 存 HttpOnly Cookie 的组合。

## 7. JWT 的常见安全问题与对策

### 问题 1：Token 泄露 → 任何人都能冒充用户

```
对策：
  ✅ 设置短过期时间（15-30 分钟）
  ✅ 使用 Refresh Token 自动续期
  ✅ 实现 Token 黑名单（登出时加入）
```

### 问题 2：Payload 可被解码 → 敏感信息泄露

```
❌ 不要在 JWT 中放密码、身份证号等敏感信息
✅ 只放用户 ID（sub）、角色等必要的非敏感信息
```

### 问题 3：密钥泄漏 → 攻击者可以伪造 JWT

```
对策：
  ✅ SECRET_KEY 用环境变量，不写在代码里
  ✅ 定期轮换密钥（JWT 支持 JWKS）
  ✅ 使用非对称加密（RS256）：私钥签名，公钥验证
```

## 8. HS256 vs RS256

| | HS256 | RS256 |
|------|-------|-------|
| **类型** | 对称加密（HMAC） | 非对称加密（RSA） |
| **密钥** | 同一个密钥签名+验证 | 私钥签名，公钥验证 |
| **适用** | 单体应用 | 微服务 |
| **优点** | 简单快速 | 更安全，适合分布式 |
| **缺点** | 密钥泄漏后果严重 | 实现稍复杂 |
