## 1. HTTPS——所有安全措施的前提

没有 HTTPS，一切都是空中楼阁。中间人可以截获所有明文传输的 Token、密码、数据。

### 开发环境启用 HTTPS

```bash

```

```python
# 或者在代码中

    import uvicorn

        "main:app",
        host="0.0.0.0",

        ssl_certfile="./cert.pem",
    )
```

### 生产环境

```nginx
# Nginx 配置 HTTPS + 反向代理
server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate     /etc/letsencrypt/live/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.example.com/privkey.pem;

    # 强制 HTTPS
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location / {
        proxy_pass http://127.0.0.1:8000;
    }
}

# 所有 HTTP 请求重定向到 HTTPS
server {
    listen 80;
    server_name api.example.com;
    return 301 https://$server_name$request_uri;
}
```

## 2. CORS 安全配置

跨域请求是 Web 安全的常见问题。FastAPI 的 `CORSMiddleware` 需要精确配置：

```python
from fastapi.middleware.cors import CORSMiddleware

# ✅ 安全的 CORS 配置
app.add_middleware(
    CORSMiddleware,
    # 精确指定允许的源——不要用 ["*"]！
    allow_origins=[
        "https://your-frontend.com",
        "https://admin.your-frontend.com",
    ],
    allow_credentials=True,  # 允许携带 Cookie
    allow_methods=["GET", "POST", "PUT", "PATCH", "DELETE"],  # 精确指定
    allow_headers=["Authorization", "Content-Type", "X-API-Key"],
    expose_headers=["X-Request-ID"],  # 前端可以读取的响应头
    max_age=600,  # 预检请求缓存时间（秒）
)
```

### CORS 配置 Checklist

| 配置项 | ❌ 危险做法 | ✅ 安全做法 |
|------|-----------|-----------|
| `allow_origins` | `["*"]` | 精确列出前端域名 |
| `allow_methods` | `["*"]` | `["GET", "POST", ...]` |
| `allow_headers` | `["*"]` | `["Authorization", ...]` |
| `allow_credentials` + `allow_origins=["*"]` | ❌ 不允许同时设置 | 用具体域名 |

## 3. 速率限制（Rate Limiting）

防止暴力破解、爬虫滥用和 DDoS 攻击。

### 简单内存版实现

```python
"""rate_limit.py — 基于内存的速率限制"""
import time
from collections import defaultdict
from fastapi import FastAPI, Request, HTTPException, status

app = FastAPI()

# ═══════ 速率限制配置 ═══════
RATE_LIMITS = {
    "default": (100, 60),      # 100 次 / 60 秒
    "/login": (5, 60),          # 登录接口：5 次 / 分（防暴力破解）
    "/register": (3, 3600),     # 注册：3 次 / 小时（防批量注册）
}

# ═══════ 存储（生产环境用 Redis） ═══════
_request_log: dict[str, list[float]] = defaultdict(list)


@app.middleware("http")
async def rate_limit_middleware(request: Request, call_next):
    """全局速率限制中间件"""
    # 获取客户端标识（用 IP，生产环境推荐用 X-Forwarded-For）
    client_ip = request.client.host

    # 获取当前路径的限制配置
    max_requests, window = RATE_LIMITS.get(
        request.url.path,
        RATE_LIMITS["default"],
    )

    # 清理过期的请求记录
    now = time.time()
    _request_log[client_ip] = [
        t for t in _request_log[client_ip]
        if now - t < window
    ]

    # 检查是否超过限制
    if len(_request_log[client_ip]) >= max_requests:
        retry_after = int(window - (now - _request_log[client_ip][0]))
        raise HTTPException(
            status_code=status.HTTP_429_TOO_MANY_REQUESTS,
            detail=f"请求太频繁，请 {retry_after} 秒后重试",
            headers={"Retry-After": str(retry_after), "X-RateLimit-Limit": str(max_requests)},
        )

    # 记录本次请求
    _request_log[client_ip].append(now)

    # 添加速率限制信息到响应头
    response = await call_next(request)
    remaining = max_requests - len(_request_log[client_ip])
    response.headers["X-RateLimit-Limit"] = str(max_requests)
    response.headers["X-RateLimit-Remaining"] = str(max(0, remaining))
    response.headers["X-RateLimit-Reset"] = str(int(now + window))
    return response
```

### 生产环境推荐

```bash

# - Nginx: limit_req_zone + limit_req
# - Traefik: RateLimit middleware
# - 云服务: AWS WAF / Cloudflare Rate Limiting
```

## 4. 安全响应头

```python
from fastapi import FastAPI
from starlette.middleware.base import BaseHTTPMiddleware


class SecurityHeadersMiddleware(BaseHTTPMiddleware):
    """为每个响应添加安全相关的 HTTP 头"""

    async def dispatch(self, request, call_next):
        response = await call_next(request)

        # 防止 MIME 类型嗅探
        response.headers["X-Content-Type-Options"] = "nosniff"

        # 防止点击劫持
        response.headers["X-Frame-Options"] = "DENY"

        # 启用浏览器 XSS 过滤
        response.headers["X-XSS-Protection"] = "1; mode=block"

        # 控制 Referrer 信息
        response.headers["Referrer-Policy"] = "strict-origin-when-cross-origin"

        # 内容安全策略（CSP）
        response.headers["Content-Security-Policy"] = (
            "default-src 'self'; "
            "script-src 'self'; "
            "style-src 'self' 'unsafe-inline'; "
        )

        # 权限策略（限制浏览器 API）
        response.headers["Permissions-Policy"] = (
            "camera=(), microphone=(), geolocation=()"
        )

        return response


app.add_middleware(SecurityHeadersMiddleware)
```

| 响应头 | 作用 | 推荐值 |
|------|------|--------|
| `X-Content-Type-Options` | 防止 MIME 嗅探 | `nosniff` |
| `X-Frame-Options` | 防止点击劫持 | `DENY` 或 `SAMEORIGIN` |
| `Strict-Transport-Security` | 强制 HTTPS | `max-age=31536000; includeSubDomains` |
| `Referrer-Policy` | 控制来源信息泄漏 | `strict-origin-when-cross-origin` |
| `Content-Security-Policy` | 防止 XSS | 根据应用定制 |

### 通用脱敏工具

```python
def sanitize_user(user: User) -> dict:
    """手动脱敏——更精细的控制"""
    return {
        "id": user.id,
        "username": user.username,
        "email": mask_email(user.email),  # zh***@example.com
        "created_at": user.created_at,
    }


def mask_email(email: str) -> str:
    """邮箱脱敏：zhang@example.com → zha**@example.com"""
    local, domain = email.split("@")
    if len(local) <= 2:
        return f"{local[0]}*@{domain}"
    return f"{local[:3]}{'*' * (len(local) - 3)}@{domain}"
```

## 7. 密码策略

```python
"""password_policy.py — 密码强度校验"""
import re
from pydantic import BaseModel, field_validator


class PasswordPolicy:
    """密码策略配置"""

    MIN_LENGTH = 8
    MAX_LENGTH = 100
    REQUIRE_UPPER = True        # 需要大写字母
    REQUIRE_LOWER = True        # 需要小写字母
    REQUIRE_DIGIT = True        # 需要数字
    REQUIRE_SPECIAL = False     # 需要特殊字符
    FORBID_USERNAME = True      # 不能包含用户名


def validate_password_strength(password: str, username: str = "") -> tuple[bool, str]:
    """验证密码强度——返回 (是否通过, 错误信息)"""
    if len(password) < PasswordPolicy.MIN_LENGTH:
        return False, f"密码至少 {PasswordPolicy.MIN_LENGTH} 位"

    if PasswordPolicy.REQUIRE_UPPER and not re.search(r"[A-Z]", password):
        return False, "密码必须包含大写字母"

    if PasswordPolicy.REQUIRE_LOWER and not re.search(r"[a-z]", password):
        return False, "密码必须包含小写字母"

    if PasswordPolicy.REQUIRE_DIGIT and not re.search(r"\d", password):
        return False, "密码必须包含数字"

    if PasswordPolicy.REQUIRE_SPECIAL and not re.search(r"[!@#$%^&*(),.?\":{}|<>]", password):
        return False, "密码必须包含特殊字符"

    if PasswordPolicy.FORBID_USERNAME and username and username.lower() in password.lower():
        return False, "密码不能包含用户名"

    return True, "密码强度合格"


# 在 Pydantic model 中使用
def create_user(user_in: UserCreate, db: Session):
    is_valid, error = validate_password_strength(user_in.password, user_in.username)
    if not is_valid:
        raise HTTPException(status_code=400, detail=error)
```

## 9. 安全审计清单

上线前逐项检查：

```
□ TLS/HTTPS 已启用，HTTP → HTTPS 自动重定向
□ CORS 配置精确到具体域名（不用 *）
□ 所有密码用 bcrypt 哈希存储
□ JWT 密钥从环境变量读取，足够长
□ JWT 设置了合理的过期时间（Access ≤ 30min, Refresh ≤ 7d）
□ 敏感接口有速率限制（login/register 等）
□ 安全响应头已配置（X-Content-Type-Options 等）
□ response_model 不暴露 password/hashed_password
□ 所有 SQL 查询经过 ORM 参数化（防注入）
□ 用户输入经过 Pydantic 验证
□ 错误信息不会泄漏内部细节（如数据库结构、堆栈信息）
□ 文件上传有大小和类型限制
□ 依赖包定期更新（pip list --outdated）
□ 生产环境关闭 echo=True 和 debug 模式
```
